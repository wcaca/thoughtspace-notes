## §0 核心原则

**3D空间是唯一真相源**——不是渲染结果，而是状态容器。

| 传统做法 | 本项目做法 |
|---|---|
| 3D只是渲染结果，状态散落在代码变量 | 3D实体携带完整状态附件，是状态容器 |
| 出问题翻代码找变量 | 出问题在3D空间中点击实体查询状态 |
| 调试靠console.log | 调试靠3D Debug Overlay + 时间线回放 |
| 单点排查，难以综合 | 状态变更链追溯因果，综合分析 |

**排查三问**:
1. **是什么**——精确表示当前状态（可量化、可可视化）
2. **为什么**——追溯状态变更因果链（根因分析）
3. **综合看**——多维度交叉验证（不孤立判断）

## §1 状态预埋：6层架构

### 1.1 第1层·实体状态附件（EntityStateAttachment）

每个3D空间实体（念头/标记/轨道控制点/锚点）自动携带完整状态信息。

```javascript
EntityStateAttachment = {
  // 身份
  id: string,
  type: 'thought'|'mark'|'orbit-point'|'anchor'|'relation',

  // 空间状态（3D中的存在）
  spatial: {
    position: {x, y, z},
    layerId: string,
    orbitParam: number,          // 轨道参数t（0-1）
    depthFromSurface: number,    // 距表面深度
    parentSpaceId: string|null,  // 父空间
  },

  // 投影状态（3D→2D映射信息）
  projection: {
    screenPosition: {x, y},      // 屏幕坐标
    projectedScale: number,      // 投影后缩放
    distortion: number,          // 该位置的扭曲度
    visibility: number,          // 可见度0-1
    opacity: number,             // 透明度
    anchorRole: 'source'|'target'|null,  // 是否作为锚点
    projectionRay: {origin, dir},  // 投影射线（排查用）
  },

  // 视角关系（相对当前视角）
  view: {
    distanceToCamera: number,
    depthContribution: number,   // 对深入度的贡献
    isInSlice: boolean,          // 是否在当前切片内
    isCenter: boolean,           // 是否是扭曲中心
    angleToCenter: number,       // 与扭曲中心夹角
  },

  // 数据状态（Yjs同步状态）
  data: {
    yjsVersion: number,
    lastUpdate: timestamp,
    syncStatus: 'synced'|'pending'|'conflict',
    source: 'user'|'sync'|'ai'|'import',
  },

  // 动画状态（瞬态，不进Yjs）
  animation: {
    type: 'phase-transition'|'view-transition'|'none',
    progress: 0-1,
    fromState: object,
    toState: object,
    startTime: timestamp,
    estimatedEnd: timestamp,
  },

  // 溯源（provenance）
  provenance: {
    createdBy: 'user'|'system'|'ai',
    createdAt: timestamp,
    lastModifiedBy: string,      // 组件名
    changeHistory: ChangeRecord[],  // 最近N条变更
  },

  // 调试
  debug: {
    renderTime: number,          // 该实体渲染耗时ms
    warnings: string[],          // 警告信息
    cacheHit: boolean,           // 渲染缓存是否命中
  },
}
```

### 1.2 第2层·3D空间状态场（SpatialStateField）

3D空间本身作为可查询的状态场，任意位置可查询。

```javascript
SpatialStateField = {
  // 32x32x32降采样网格
  grid: Map<cellId, CellState>,

  CellState = {
    position: {x, y, z},
    depthValue: number,          // 该位置深入度
    distortion: number,          // 该位置扭曲度
    thoughtDensity: number,      // 念头密度
    nearestThought: string,      // 最近的念头id
    visibleFromCamera: boolean,
    layerId: string,
    orbitParam: number,          // 轨道参数
  },

  // 查询接口
  query(point): CellState,
  queryRay(origin, dir): RayState,    // 射线查询
  queryRegion(bounds): RegionState,   // 区域查询
  queryLayer(layerId): LayerState,    // 层查询
}
```

### 1.3 第3层·时间线快照（TimelineSnapshots）

每100ms或关键事件拍摄快照，支持时间旅行调试。

```javascript
Snapshot = {
  timestamp, frame,

  // 视角状态
  view: {
    cameraPos, cameraTarget,
    orbitParam, viewConfigId,
    distortion, depthValue,
    orbitPathId,
  },

  // 所有实体状态摘要（只存关键字段，控制内存）
  entities: Map<entityId, EntityStateSummary>,

  // 性能指标
  performance: {
    frameTime, depthTime, projectionTime, renderTime,
    cacheHitRate,
  },

  // 触发源
  userAction: UserAction | null,
  yjsChanges: YjsChange[] | null,

  // 状态变更链引用
  changeChainHead: string|null,  // 本帧首个变更id
}
```

**存储策略**:
- 环形缓冲，保留最近1000个快照（约100秒）
- 关键事件（用户操作、Yjs变更、错误）强制快照
- 开发环境保留更多，生产环境精简

### 1.4 第4层·状态变更链（StateChangeChain）

每次状态变更记录完整因果关系。

```javascript
StateChangeRecord = {
  id, timestamp,

  // 来源
  source: {
    type: 'user-action'|'yjs-sync'|'animation'|'system',
    action: UserAction | YjsChange,
    component: string,          // 触发组件
    stackTrace: string,         // 调用栈
  },

  // 影响
  effects: {
    yjsUpdates: YjsUpdate[],
    transientUpdates: TransientUpdate[],
    cacheInvalidations: string[],  // 失效的缓存key
    renderRequests: RenderRequest[],
  },

  // 因果链
  causedChanges: string[],     // 导致的其他变更id
  causedBy: string|null,       // 被哪个变更导致

  // 状态快照（变更前后）
  beforeState: object,
  afterState: object,
}
```

### 1.5 第5层·3D Debug Overlay（空间内可视化调试）

在3D空间中直接可视化所有状态，可切换显示层。

| Overlay层 | 可视化内容 | 用途 |
|---|---|---|
| **深入度热力图** | 红色=高深入度，蓝色=低 | 排查内窥镜视角问题 |
| **投影射线** | 每条3D→2D的投影线 | 排查卡片位置问题 |
| **扭曲度等值线** | 扭曲度相等的连线 | 排查扭曲过渡问题 |
| **轨道路径** | 控制点+曲线+当前位置 | 排查视角轨道问题 |
| **标记系统** | 所有标记的位置和类型 | 排查标记系统问题 |
| **层边界** | 6层+2外置的3D边界 | 排查层归属问题 |
| **缓存命中区** | 绿色=命中，红色=失效 | 排查性能问题 |
| **念头密度场** | 密度高=亮，密度低=暗 | 排查布局问题 |
| **Yjs同步状态** | 同步=绿，待同步=黄，冲突=红 | 排查数据问题 |
| **动画状态** | 进行中的动画高亮 | 排查动画冲突 |

### 1.6 第6层·状态查询语言（SpatialQuery）

统一接口查询任意状态。

```javascript
// 查询接口
SpatialQuery
  .at(point)                               // 某位置的状态
  .entity(id)                              // 某实体的完整状态附件
  .ray(origin, direction)                  // 射线穿过的所有状态
  .region(bounds)                          // 某区域内所有状态
  .layer(layerId)                          // 某层的所有状态
  .timeline(t1, t2)                        // 某时间段的状态变更
  .causedBy(actionId)                      // 某操作导致的所有变更
  .where(state => state.distortion > 0.5)  // 条件查询
  .diff(snapshot1, snapshot2)              // 两个快照的差异
  .trace(entityId, field)                  // 某字段的历史变更轨迹
```

## §3 综合排查：多维度交叉分析

### 3.1 综合排查矩阵

排查不只看单一维度，要**多维度交叉验证**。

| 问题现象 | 可能维度 | 排查路径 |
|---|---|---|
| 卡片位置错误 | 投影·扭曲度·视角·轨道 | 查投影射线→查扭曲度→查视角配置→查轨道参数 |
| 卡片大小不对 | 3D体积·displayScale·投影缩放·字数 | 查体积公式→查displayScale→查投影缩放→查字数 |
| 卡片数量不对 | 层筛选·切片·可见度·性能限制 | 查层筛选规则→查切片厚度→查可见度→查性能限制 |
| 空间结构错乱 | 层定义·标记系统·轨道·认知框架 | 查层定义→查标记实例→查轨道→查框架 |
| 视角异常 | 相机参数·轨道·深入度·扭曲度 | 查相机→查轨道参数→查深入度→查扭曲度 |
| 切换动画卡顿 | 帧调度·缓存·动画冲突·性能 | 查帧调度→查缓存命中→查动画冲突→查性能 |
| 扭曲效果异常 | 深入度·射线累积·shader·中心 | 查深入度→查射线→查shader→查扭曲中心 |
| 内窥镜不触发 | 深入度阈值·轨道段·视角配置 | 查阈值→查轨道段类型→查视角配置 |
| 投影锚点错位 | 标记线·投影·卡片位置·扭曲度 | 查标记线3D→查投影→查卡片位置→查扭曲度 |
| 数据不同步 | Yjs·缓存·瞬态层·同步状态 | 查Yjs版本→查缓存失效→查瞬态层→查同步状态 |

### 3.2 综合排查工作流

```
发现问题
    │
    ▼
Step 1: 精确描述问题
    ├─ 是什么现象？
    ├─ 哪个实体/位置/视角？
    ├─ 何时发生？（时间点/操作）
    └─ 期望vs实际？
    │
    ▼
Step 2: 3D空间直接查询
    ├─ 点击问题实体 → 查看EntityStateAttachment
    ├─ 查SpatialStateField → 该位置的状态
    └─ 开启Debug Overlay → 可视化异常
    │
    ▼
Step 3: 时间线回放
    ├─ 定位问题时间点
    ├─ 回放前后N帧
    └─ 识别触发源（用户操作/Yjs变更/动画/系统）
    │
    ▼
Step 4: 状态变更链追溯
    ├─ 从触发源出发
    ├─ 沿causedChanges追下游影响
    ├─ 沿causedBy追上游根因
    └─ 识别责任组件
    │
    ▼
Step 5: 多维度交叉验证
    ├─ 按§3.1矩阵列出可能维度
    ├─ 逐一验证各维度状态
    ├─ 交叉比对（A维度正常但B维度异常）
    └─ 识别真正异常维度
    │
    ▼
Step 6: 期望值对比
    ├─ 用ExpectedCalculator计算期望值
    ├─ StateDiff比较期望vs实际
    ├─ 识别偏差字段
    └─ 评估偏差严重度
    │
    ▼
Step 7: 根因确认与修复
    ├─ 确认根因（哪个组件/哪个计算）
    ├─ 修复
    ├─ 用时间线快照验证修复
    └─ diff修复前后快照
```

### 3.3 排查示例：卡片位置错误

**问题**：念头A的卡片显示在屏幕左上角，但期望在中央。

```
Step 1: 精确描述
  - 实体：念头A (id: thought-001)
  - 现象：显示在屏幕(100,100)，期望(400,300)
  - 时间：13:00:01.500
  - 操作：用户切换视角

Step 2: 3D空间查询
  点击念头A → EntityStateAttachment:
    spatial.position = {2, 1, 3}  ✓ 正确
    projection.screenPosition = {100, 100}  ✗ 错误
    projection.distortion = 0.2  ✗ 异常（应为0.6）
    projection.projectionRay.dir = {0.1, 0.9, 0.4}  ✗ 方向异常

Step 3: 时间线回放
  13:00:01.000 位置正确，distortion=0.6
  13:00:01.234 用户切换视角 → view-transition启动
  13:00:01.500 投影重算，但distortion=0.2  ← 问题出现

Step 4: 状态变更链
  view-transition (13:00:01.234)
    ├─ causedBy: user-action "swipe"
    ├─ effects:
    │   ├─ transientUpdates: [viewTransition.progress=0.5]
    │   ├─ cacheInvalidations: ['projection', 'distortion']
    │   └─ renderRequests: [frame-1001]
    └─ causedChanges: [projection-recompute]

  projection-recompute (13:00:01.500)
    ├─ causedBy: view-transition
    ├─ effects:
    │   ├─ transientUpdates: [thought-001.projection={100,100}]
    │   └─ 但distortion未更新（仍为0.2）
    └─ 根因：FrameScheduler将distortion更新推迟到下一帧

Step 5: 多维度交叉验证
  ├─ 投影维度：异常（distortion过期）
  ├─ 视角维度：正常（相机位置正确）
  ├─ 轨道维度：正常（轨道参数正确）
  ├─ 数据维度：正常（Yjs同步）
  └─ 动画维度：正常（动画进行中）
  → 真正异常：投影维度（distortion与projection不同步）

Step 6: 期望值对比
  ExpectedCalculator.distortion(depth=0.8) = 0.6
  实际 distortion = 0.2
  StateDiff: delta=-0.4, severity=critical
  期望位置 = (400, 300)
  实际位置 = (100, 100)
  StateDiff: delta={-300,-200}, severity=critical

Step 7: 根因确认与修复
  根因：FrameScheduler调度问题，distortion更新被推迟
  修复：确保projection和distortion在同一帧更新
  验证：修复后快照对比，distortion与projection同步
```

### 3.4 排查示例：内窥镜不触发

**问题**：用户视角深入到圆内部，但内窥镜效果未出现。

```
Step 1: 精确描述
  - 位置：轨道参数t=0.8（应为deep段）
  - 现象：distortion=0，期望distortion>0.5
  - 时间：13:00:05.000

Step 2: 3D空间查询
  查SpatialStateField.at(cameraPos):
    depthValue = 0.15  ✗ 低于阈值0.2
    thoughtDensity = 0.3
    nearestThought = thought-005

  开启深入度热力图Overlay:
    相机位置附近为蓝色（低深入度）
    但圆内部有红色区域（高深入度）在远处

Step 3: 时间线回放
  13:00:04.000 用户开始沿轨道深入
  13:00:04.500 相机进入transition段
  13:00:05.000 相机到达deep段，但depthValue仍=0.15

Step 4: 状态变更链
  orbit-move (13:00:04.000)
    └─ effects: cameraPos更新

  depth-recompute (每4帧)
    ├─ 射线累积法计算
    ├─ 但射线方向只取屏幕中心单条
    └─ 未命中任何念头 → depth=0

  根因：射线累积法采样不足，只取屏幕中心射线
  实际：相机在圆内部，但屏幕中心方向恰好无念头

Step 5: 多维度交叉验证
  ├─ 深入度维度：异常（采样不足）
  ├─ 轨道维度：正常（t=0.8在deep段）
  ├─ 视角维度：正常（相机在圆内部）
  └─ 扭曲度维度：异常（因深入度低，未触发）

Step 6: 期望值对比
  ExpectedCalculator.distortion(depth=0.8) = 1.0
  但实际depth=0.15（采样不足导致）
  → 问题在depth计算，不在distortion映射

Step 7: 根因确认与修复
  根因：射线累积法采样不足，单条射线易漏检
  修复：改用32x32网格采样或多射线平均
  验证：修复后depth=0.85，distortion=0.95，内窥镜触发
```

## §4 自动诊断系统

### 4.1 诊断规则引擎

```javascript
DiagnosticRule = {
  id: 'D001',
  name: '卡片位置偏差检测',
  trigger: 'every-frame' | 'on-user-action' | 'on-error',

  condition: (state) => {
    for (const entity of state.entities) {
      const expected = ExpectedCalculator.cardPosition(entity, state.viewConfig);
      const actual = entity.projection.screenPosition;
      const delta = distance(expected, actual);
      if (delta > 10) return { entity, expected, actual, delta };
    }
    return null;
  },

  report: (match) => ({
    severity: 'critical',
    message: `念头${match.entity.id}位置偏差${match.delta}px`,
    suggestion: '检查投影计算和扭曲度同步',
    entities: [match.entity.id],
    relatedChanges: traceRecentChanges(match.entity.id, 'projection'),
  }),
}
```

## §5 实现方式

### 5.1 自动预埋（装饰器/代理模式）

```javascript
// 所有3D实体自动携带状态附件
class SpatialEntity {
  constructor() {
    this.__state = new EntityStateAttachment(this);
  }
}

// 所有状态变更自动记录
@traceable
class Thought extends SpatialEntity {
  @recordChange(reason='position-update')
  setPosition(x, y, z) {
    this.position = {x, y, z};
    // 自动记录：who/when/why/before/after
  }
}

// 所有渲染自动更新状态附件
class RenderPipeline {
  frame() {
    for (const entity of this.entities) {
      entity.__state.updateProjection(this.camera);
      entity.__state.updateViewRelation(this.camera);
    }
    this.snapshotManager.maybeSnapshot();
  }
}
```

### 5.2 性能策略

| 策略 | 说明 | 开销 |
|---|---|---|
| 采样快照 | 每100ms或关键事件快照，非每帧 | <1% |
| 脏标记 | 只更新变化的实体状态附件 | <2% |
| 降采样状态场 | 32x32x32网格 | <1ms/帧 |
| 懒求值 | debug信息只在开启Overlay时计算 | 0%（关闭时） |
| 环形缓冲 | 时间线只保留最近1000个快照 | 固定内存 |
| 生产环境降级 | 关闭provenance详细记录 | <0.5% |

**总开销估计**：开发环境3-5%，生产环境<1%

### 5.3 查询接口实现

```javascript
// 统一查询入口
class SpatialQuery {
  at(point) { return this.field.query(point); }
  entity(id) { return this.entities.get(id)?.__state; }
  ray(origin, dir) { return this.field.queryRay(origin, dir); }
  region(bounds) { return this.field.queryRegion(bounds); }
  layer(layerId) { return this.field.queryLayer(layerId); }
  timeline(t1, t2) { return this.snapshots.range(t1, t2); }
  causedBy(actionId) { return this.changeChain.trace(actionId); }
  where(predicate) { return [...this.entities.values()].filter(predicate); }
  diff(s1, s2) { return StateDiff.compare(s1, s2); }
  trace(entityId, field) { return this.changeChain.traceField(entityId, field); }
}
```
