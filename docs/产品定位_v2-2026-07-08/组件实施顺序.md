|---|---|---|---|---|

| **0.2** | `runtime/flags/v2-migration.js` | runtime | 无 | feature flag：`v2.enabled`（默认false） |

- 所有实体状态可序列化为JSON（供AI读取）
- 状态变更链可导出（供AI分析因果）
- 诊断报告可生成（供AI自动检测问题）
- 时间线快照可回放（供AI回溯问题）

## §2 S1 · 空间本体（18个组件）

> **设计原则**: 6层默认结构只是标记系统的一种预设实例，并非固定物理结构。标记系统（mark-system）是层/参考线/阻尼/容器边界的统一抽象基础，与坐标系、视角能力三方联动。
> **排查基础**: S1阶段填充S0建立的SceneStateStore骨架，完成状态预埋6层架构的5个core组件。

| 序号 | 组件 | 层 | 依赖 | 说明 |
|---|---|---|---|---|
| **1.1** | `core/space.js` | L1 | 无 | 有限3D晶体边界计算、三维度定义 |
| **1.2** | `core/mark-system.js` | L1 | 1.1 | **标记系统抽象**（MarkType注册/实例化/查询），层/参考线/阻尼/容器边界都是其预设实例。对应设计§4空间组织器 |
| **1.3** | `core/layer.js` | L1 | 1.2 | 6层+2外置层系统、近大远小——**改为基于mark-system的预设实例化**（非硬编码） |
| **1.4** | `core/cognitive-framework.js` | L1 | 1.3 | 8种框架定义（6层+2外置/三网络/曼陀罗/原型/脑区/八维/生命之树/自定义）——框架=标记系统的预设组合 |
| **1.5** | `core/view-orbit.js` | L1 | 1.1 | 圆周轨道视角数学（纯逻辑，无Three.js） |
| **1.6** | `render/crystal-space.js` | L2 | 1.1 | 晶体空间渲染（边界、星尘背景） |
| **1.7** | `render/layer-renderer.js` | L2 | 1.3, 1.6 | 层渲染（层间距、近大远小）——消费mark-system实例 |
| **1.8** | `render/view-orbit-camera.js` | L2 | 1.5 | 圆周轨道相机（消费core/view-orbit数学） |
| **1.9** | `render/base-plane.js` | L2 | 1.6, 1.7 | 3/4基准面（3D扭曲投影，非物理切片） |
| **1.10** | `render/operation-zone.js` | L2 | 1.9 | 1/4操作区（外层环绕、3D缩略图） |

| **1.14** | `core/spatial-state-field.js` | L1 | S0.6 | **排查基础**：3D空间状态场（32³网格+查询接口），任意位置可查询深入度/扭曲度/密度 |
| **1.15** | `core/state-snapshot.js` | L1 | S0.6 | **排查基础**：时间线快照管理（环形缓冲1000个，每100ms或关键事件快照） |
| **1.16** | `core/state-change-chain.js` | L1 | S0.6 | **排查基础**：状态变更链（causedBy/causedChanges双向追溯，记录完整因果） |
| **1.17** | `core/diagnostic-engine.js` | L1 | 1.16 | **排查基础**：诊断规则引擎（15条内置规则D001-D015，自动检测+综合关联分析） |
| **1.18** | `core/spatial-query.js` | L1 | 1.14, 1.15, 1.16 | **排查基础**：统一查询语言（at/entity/ray/region/layer/timeline/causedBy/where/diff/trace） |

**并行机会**:
- 1.1完成后，1.2/1.5可并行
- 1.2完成后，1.3启动；1.3完成后1.4启动
- 1.6完成后，1.7/1.8/1.11可并行
- 1.9完成后，1.10/1.12/1.13可并行
- 1.14/1.15/1.16在S0.6完成后可并行（均依赖SceneStateStore骨架）
- 1.17依赖1.16；1.18依赖1.14/1.15/1.16

**关键路径**: 1.1 → 1.2 → 1.3 → 1.7 → 1.9 → 1.10

**三方联动约束**: 标记系统(mark-system) ↔ 坐标系(space/coordinate) ↔ 视角能力(view-orbit)三者必须可联动——改标记自动反映到视角可选范围，改视角可筛选可见标记。

**AI自排查能力约束**: S1完成的5个排查基础组件必须让AI能够：
- 读取任意实体的完整状态（SpatialQuery.entity(id)）
- 查询任意位置的状态场（SpatialQuery.at(point)）
- 回放任意时间段（SpatialQuery.timeline(t1,t2)）
- 追溯任意变更的因果（SpatialQuery.causedBy(actionId)）
- 获取自动诊断报告（DiagnosticEngine.report()）
- 比较期望与实际（SpatialQuery.diff(expected, actual)）
