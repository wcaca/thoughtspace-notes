## §0 设计灵魂

### 0.1 一句话定位

**用户手里捧着的不是一个"工具"，是一个"可塑的拓扑流形"——它的念头田野的物理形态。用户通过把玩它来觉察自己的思维模式。**

### 0.2 产品哲学

> 用户不是在"使用"一个工具管理任务，而是在"照料"一片属于自己的念头田野。
> 记录是播种，整理是耕作，观察是静观其生长与凋零，行动是收获。

### 0.3 自我觉察飞轮（v2 灵魂）

```
       ┌─────────────────────────────────────┐
       │                                     │
       │  1. 用户在空间中记录念头(即时性)    │
       │           ↓                          │
       │  2. 空间形成拓扑结构(整体性)         │
       │           ↓                          │
       │  3. 用户从拓扑结构中觉察盲区/隐喻    │
       │           ↓                          │
       │  4. 用户写"关于这个觉察"的念头/笔记 │
       │           ↓                          │
       │  5. 新念头融入空间拓扑结构           │
       │           ↓                          │
       └────── (回到 1,飞轮持续) ────────────┘
```

**飞轮不可中断的约束**:
- ❌ 不能"AI 自动提醒用户去觉察" → 违背"提示而非执行"
- ❌ 不能"用数值展示用户盲区" → 违背"不评分"
- ✅ 必须"通过空间拓扑关系**呈现**,让用户**自我觉察**"
- ✅ 用户觉察后"**自然**写一个关于这个觉察的念头/笔记"

### 0.4 四种手感的本质整合

| 手感 | 本质 | 在系统中的角色 |
|---|---|---|
| **3维空间** | 容器与拓扑 | 所有内容的本体存在——有限3D"念头晶体" |
| **魔方操作** | 有结构的空间变换 | 视角沿轨道转动，切换看的面 |
| **消消乐平面** | 2D面上的即时反馈 | 当前面上拖拽/合并/拆分，60fps顺滑 |
| **念头/记忆切换** | 同一实体的相变 | 念头（轻）↔ 记忆（重）的形态变换 |

**整合点**: 面是投影，3D是本体。用户永远在看一个2D面（基准面，3/4屏），但这个面是3D空间的一个切片。所有操作在2D面上进行（消消乐式），操作的实际效果在3D中（魔方式空间变换），念头和记忆是3D空间中的实体，可以相变。

## §1 空间本体

### 1.1 有限3D"念头晶体"

**核心约束**: 空间不是无限宇宙，是一个有限3D体——像捧着一个东西。

| 特征 | 说明 |
|---|---|
| **有限边界** | 空间有明确边界，不是无限宇宙。用户的念头田野是"有限的可把玩对象" |
| **念头紧致堆积** | 念头不是稀疏漂浮，是紧致堆积，互相有引力，填满空间 |
| **念头大小可观** | 每个念头占空间的1/50到1/100（不是小点），有实体感 |
| **整体可旋转** | 捧着整个3D体旋转，看不同面 |
| **可进入子空间** | 捏合某个念头，进入它的内部子空间（分形嵌套） |
| **面是切片** | 基准面是3D体的切片，看到这一层的念头 |

**为什么不是无限空间**: 无限3D空间里漂浮小点 = 数据可视化思路，不是把玩思路。用户需要感觉"手里捧着一团念头"，能看清，能把玩，空间利用率高。

### 1.2 默认结构: 6层3D主体 + 2外置层

```
         ╭─────────────────╮
        ╱   外置·顶层       ╲   ← 默认名称"规则"
       │   （3D空间，可记录） │      外部规则（信念难改变）
        ╲                   ╱
         ╰─────────────────╯

         ╭─────────────────╮
        ╱   显意识·表层（大）╲     ┐
       │   显意识·中层         │    │ 6层
       │   显意识·深层         │    │ 3D主体
       │  ─── 分界线 ───       │    │ 近大远小
       │   潜意识·深层         │    │
       │   潜意识·中层         │    │
        ╲   潜意识·表层（小）╱     ┘
         ╰─────────────────╯

         ╭─────────────────╮
        ╱   外置·底层       ╲   ← 默认名称"结构"
       │   （3D空间，可记录） │      外部物质/结构
        ╲                   ╱
         ╰─────────────────╯
```

**关键设计**:
- **顶层外置**: 也是完整3D空间（念头晶体），可在里面记录念头/记忆，默认名称"规则"，记录外部规则（信念难改变的部分）
- **底层外置**: 也是完整3D空间（念头晶体），可在里面记录念头/记忆，默认名称"结构"，记录外部物质/结构（信念难改变的部分）
- **顶层底层名称可自定义**: 用户可改名称
- **6层3D主体**: 3层显意识 + 3层潜意识，垂直堆叠
- **近大远小**: 越接近视角起点的层越大、可存在数量越多

### 1.3 三维度空间结构

3D主体有三个维度，各维度独立可自定义:

```
维度1: 上下（垂直）
   - 6层堆叠（3显意识+3潜意识）
   - 近大远小
   - 顶层底层外置

维度2: 内外（径向）
   - 外层 = 表层念头
   - 内层 = 朝向圆心，自我信念
   - 内层被外层覆盖，需自定义视角看
   - 单个念头有内外结构，形成区块时显现

维度3: 圆周（轨道）
   - 视角沿圆周表面转动
   - 左右划切换视角位置
   - 不同位置看到不同的左右内容
   - 轨道形状不规则（默认），可自定义
```

**关键约束**: 轨道形状≠内容位置。不能因为轨道是圆形/不规则形就判定内容必须在某个区域。潜意识自定义与轨道自定义不混淆。

### 1.4 默认视角轨道（不规则，可改）

视角沿圆周表面转动，不同位置看到不同的左右内容组合:

| 视角位置 | 左边 | 右边 | 划动方式 |
|---|---|---|---|
| **初始** | 过去记忆（用户输入） | 行为模式（信念→思维→情绪→念头，用户排序） | 默认 |
| **右划1** | 念头区 | Todo | 切到工作模式 |
| **右划2** | Todo | 计划+未做 | 切到计划模式 |
| **右划3** | 计划+未做 | 外部物质/结构 | 切到物质层 |
| **左划** | 过去记忆 | 行为模式 | 切到回溯模式 |

**行为模式排序（内层层级，可重排）**:
```
外层 ← 念头 → 情绪 → 思维 → 信念 → 内层（圆心）
```

### 1.5 空间外部信息边界

3D空间（包括顶层和底层外置层）外部存在用户能感知但无法直接操作的信息边界。这些外部信息是用户尚未纳入念头田野的内容，作为空间的背景存在。

### 1.6 可切换认知框架

不绑死任何一种理论，提供可切换框架系统:

| 框架 | 分区数 | 适用场景 |
|---|---|---|
| **6层+2外置**（默认） | 8 | 通用，显意识/潜意识分层 |
| 三网络（DMN/CEN/SN） | 3 | 简洁，脑科学实证 |
| 曼陀罗（方向+中心） | 5 | 方向感强，自我探索 |
| 原型（阴影/人格面具/自性） | 4-7 | 深度心理探索 |
| 脑区（前额叶/海马/杏仁核等） | 5 | 功能定位 |
| 八维（Ne/Ni/Se/Si/Te/Ti/Fe/Fi） | 8 | 认知功能细分 |
| 生命之树（10原质+22路径） | 10+ | 复杂思考 |
| 自定义 | 用户定义 | 用户自建 |

## §2 可定制系统（核心特色）

### 2.1 可定制性边界

**用户可自定义**:

| 自定义项 | 默认 | 可改 |
|---|---|---|
| **层数** | 6层 | 可增减 |
| **层定义** | 3显意识+3潜意识 | 每层含义 |
| **层归属** | 创建位置决定 | 念头在哪一层 |
| **滑动轨道形状** | 不规则 | 可改 |
| **视角切换方案** | 默认方案 | 每视角独立滑动方案 |
| **潜意识位置** | 背面 | 可改 |
| **轨道位置** | 圆周 | 可改 |
| **内层层级** | 信念→思维→情绪→念头 | 可重排 |
| **参考线** | 默认 | 用户自定义 |
| **阻尼组件** | 底层/背面 | 位置、阻尼感 |
| **认知框架** | 6层+2外置 | 可切换 |
| **念头显示大小** | 系统计算 | 用户手动调整（独立于3D体积，影响2D显示） |
| **念头颜色** | 温度映射 | 用户指定任意颜色 |
| **念头视觉材质** | 默认 | 金属/玻璃/木质/水晶/烟雾/水滴等 |
| **念头触动声音** | 无 | 自然音/乐器音/人声/环境音等 |
| **念头关系规则** | 默认提示合并 | 合并/互斥/并列/包含/被包含的规则算法 |
| **规则算法** | 默认 | 图形化配置/规则算法/AI辅助编写 |

**不可自定义**:

| 不可变项 | 原因 |
|---|---|
| **3D有限空间** | 核心哲学（念头晶体） |
| **念头/记忆相变** | 核心交互 |
| **嵌套子空间** | 分形结构 |
| **空间组织器统一性** | 参考系+标记+容器统一 |
| **觉察方式** | 标记+空间拓扑呈现 |

### 2.2 可定制性的原则

1. **各维度独立**: 潜意识自定义与轨道自定义不混淆
2. **视角决定滑动方案**: 切换到某视角后，滑动行为由该视角的方案决定
3. **轨道形状≠内容位置**: 不能因为轨道形状判定内容所在区域
4. **不破坏界面统一性**: 所有自定义都通过统一界面操作，不增加界面复杂度

## §4 空间组织器（统一系统）

### 4.1 三系统统一

参考系系统、标记系统、容器/分区系统是同一个系统的三种表现:

```
用户画一个面（标记）
  → 这个面把空间分成两个区域（容器/分区）
  → 这个面定义了一个新的坐标参考（参考系）
  → 这个面可以作为旋转轴（标记功能）
  → 这个面可以贴上阻尼组件（容器功能）

用户选择6层分区（参考系）
  → 6个区域就是容器（容器/分区）
  → 区域边界就是标记（标记系统）

阻尼组件（容器）
  → 它的位置定义了"正面/背面"（参考系）
  → 它的边界是标记（标记系统）
  → 它是一个容器（容器/分区）
```

**核心概念**: 任何参考系同时也是容器和标记。任何容器同时也是参考系和标记。任何标记同时也是参考系和容器。这三者统一成"空间组织器"。

### 4.2 阻尼组件

阻尼组件是"空间组织器"的一个特殊实例:

| 属性 | 说明 |
|---|---|
| **本质** | 可自定义放置的组件，承载该空间的"盲点或阴暗面" |
| **默认位置** | 底层/背面（显意识的反面） |
| **内容** | 混合——空容器部分 + 自动收集部分（被忽视/被压抑的念头） |
| **阻尼感** | 物理+视觉，可自定义 |
| **嵌套规则** | 整个意识空间默认放置；子空间不默认放置（但用户可选择） |
| **自定义** | 用户可以移动它的位置 |

### 4.3 标记功能

用户可以在空间上画线/面/体作为标记:

| 标记类型 | 用途 |
|---|---|
| **线** | 作为旋转轴（层旋转）、作为参考线 |
| **面** | 作为切片、作为分区边界 |
| **体** | 作为封闭空间、作为容器 |

## §5 念头实体

### 5.1 念头的3D体外观

念头不是纯几何体，是"可读的3D卡片体"——上面有文字，字体大小有下限。

**基础形态: 可塑多面体**

念头的3D体不是固定形状，是由属性决定的可塑多面体。分为**系统映射属性**（根据念头状态自动计算）和**用户自定义属性**（用户主动设置）两类:

**系统映射属性**（自动）:

| 属性 | 映射到3D体 | 效果 |
|---|---|---|
| 类型（念头/记忆） | 多面体类型 | 念头=锐利低面数（四面体/立方体）；记忆=圆润高面数（球体/高面数多面体） |
| 温度 | 颜色 | 暖色=活跃（金/橙）；冷色=沉寂（深蓝/紫） |
| 活跃度 | 透明度 | 活跃=不透明；沉寂=半透明 |
| 年龄 | 旋转速度 | 新念头=快旋转；老念头=慢旋转/静止 |
| 关联度 | 发光强度 | 关联多=发光强；孤立=暗淡 |

**用户自定义属性**（主动）:

用户可以对念头的3D体进行完全自定义，覆盖系统默认值:

| 自定义项 | 说明 | 入口 |
|---|---|---|
| **显示大小** | 用户可捏合/拉伸手动调整（0.5-3.0倍），影响2D基准面上的卡片显示尺寸，独立于3D体积 | 直接手势操作 / 念头菜单 |
| **颜色** | 用户可指定任意颜色，覆盖温度映射的默认色 | 念头菜单 → 外观 → 颜色 |
| **视觉材质** | 用户可选择不同材质（金属/玻璃/木质/水晶/烟雾/水滴等），影响反射、折射、表面纹理 | 念头菜单 → 外观 → 材质 |
| **触动声音** | 用户可为念头绑定触动声音（自然音/乐器音/人声/环境音等），点击或拖动念头时播放 | 念头菜单 → 外观 → 声音 |

**自定义优先级**: 用户自定义属性 > 系统映射属性。用户未自定义时使用系统默认，自定义后以用户设置为准。

**自定义的意义**:
- **显示大小**: 用户调整2D基准面上的卡片显示尺寸，影响可显示数量（手动放大=显示更大但数量少，手动缩小=显示更小但数量多）
- **颜色**: 用户自定义语义编码（如红色=紧急，蓝色=冷静，绿色=成长）
- **视觉材质**: 用户表达念头的"质感"（金属=坚硬固执，水滴=流动变化，水晶=通透洞察）
- **触动声音**: 用户为念头赋予听觉记忆（重要的念头有独特声音，便于辨识）

### 5.2 念头/记忆的内外结构

| 状态 | 内外结构 |
|---|---|
| 单个念头/记忆 | 有内外结构（表面内容+深层含义），但外面看不出来 |
| 念头/记忆形成区块 | 内外结构显现，需要呈现 |
| 封闭空间 | 内层可见，接近自我信念 |

**内层念头/记忆**: 朝向圆心，接近自我信念、对自身思维情绪感知。被外层覆盖，需要自定义视角才能看到。

**空间坐标系标记的适用范围**: 念头和记忆都可使用空间坐标系标记。空间组织器（参考系+标记+容器）对念头和记忆一视同仁——任何可在念头上做的空间标记、坐标定义、区域划分，同样可在记忆上做。

### 5.3 相变（念头↔记忆）

念头和记忆是同一实体的两种相态，可以连续相变:

```
念头（轻，一句话）
  ↓ 长按 → 展开
中间态（八面体，渐变）
  ↓
记忆（重，图文混合笔记）

记忆（重）
  ↓ 长按 → 折叠
中间态
  ↓
念头（轻）
```

**相变的连续性**:
- 用一个进度值t（0=念头，1=记忆），渲染时插值
- 相变分两阶段——先放大卡片（几何相变），再加载内容（内容相变）
- 折叠/展开是流畅动画，不是页面跳转

### 5.4 占据空间比例

**大小决定因素**:
```
体积 = 基础大小 × 内容系数 × 关联系数 × 温度系数

基础大小：固定值（保证最小可见性）
内容系数：一句话=1.0；短段=1.5；长文=2.0；图文=2.5
关联系数：每多一个关联 +0.1（引力效应，上限2.0）
温度系数：活跃=1.2；中性=1.0；沉寂=0.7
```

**空间比例约束**:
- 最小体积：基础大小的0.5倍（防止消失）
- 最大体积：基础大小的5倍（防止吞噬其他）
- 念头密集区：自动避让（力导向）
- 念头稀疏区：自然膨胀
- **字体下限**: 保证可读，不能太小
- **系数乘积裁剪**: 体积公式各系数乘积若超出[0.5, 5.0]倍基础大小范围，先计算乘积再裁剪到约束范围内（min(max(乘积, 0.5), 5.0)）

**温度取值范围**: 温度值为0-1的浮点数。0=完全沉寂，1=完全活跃。温度衰减随时间进行（具体衰减算法可由用户配置规则）。

### 5.5 相变时的空间重组

当念头展开为记忆时（占据空间变大）:

```
状态1：3个念头，大小均匀
┌──────┐ ┌──────┐ ┌──────┐
│念头A │ │念头B │ │念头C │
└──────┘ └──────┘ └──────┘

状态2：念头B展开为记忆（变大）
┌──────┐ ┌──────────────┐ ┌──────┐
│念头A │ │  记忆B        │ │念头C │
│(缩小)│ │  图文混合     │ │(缩小)│
└──────┘ └──────────────┘ └──────┘

状态3：自动重组后
┌────┐ ┌──────────────┐ ┌────┐
│A   │ │  记忆B        │ │C   │
│    │ │  展开内容     │ │    │
└────┘ └──────────────┘ └────┘
其他念头缩小，但字体仍可读，关系仍清晰
```

**重组原则**:
- 其他念头自动缩小，给记忆让出空间
- 字体有下限，保证可读
- 关系理解不受影响
- 空间利用率高（空间被内容填满）

### 5.6 念头关系与规则系统

念头和记忆之间存在多种结构关系，用户可以设定规则来定义这些关系如何运作。

#### 5.6.1 五种基础关系

| 关系 | 符号 | 含义 | 视觉表现 |
|---|---|---|---|
| **合并** | A + B → C | 两个念头合并为一个新念头 | 两体融合动画 |
| **互斥** | A ⊥ B | 两个念头不能同时存在于同一区域 | 靠近时斥力/警告色 |
| **并列** | A ∥ B | 两个念头平级共存 | 等距并排 |
| **包含** | A ⊃ B | A包含B（B是A的子念头） | B在A内部 |
| **被包含** | A ⊂ B | A被B包含（A是B的子念头） | A在B内部 |

#### 5.6.2 规则系统

用户可以为关系设定规则。规则是**算法**，决定关系如何触发和执行:

**规则的结构**:
```
规则 = {
  触发条件: 何时检查此规则
  判定逻辑: 满足什么条件时关系成立
  执行动作: 关系成立时做什么
  优先级: 多条规则冲突时的处理顺序
}
```

**规则示例**:

```
规则1: 相同标签自动合并
  触发条件: 用户为两个念头打上相同标签
  判定逻辑: tag(A) == tag(B)
  执行动作: 提示用户"是否合并？"，用户确认后合并
  优先级: 中

规则2: 互斥念头自动分离
  触发条件: 两个互斥念头距离 < 阈值
  判定逻辑: distance(A, B) < threshold && isMutex(A, B)
  执行动作: 力导向施加斥力，自动分离
  优先级: 高

规则3: 包含关系自动布局
  触发条件: A ⊃ B 成立
  判定逻辑: parent(B) == A
  执行动作: B的3D体渲染在A内部，A的大小自适应容纳B
  优先级: 中

规则4: 温度衰减自动沉层
  触发条件: 念头温度 < 0.3 持续7天
  判定逻辑: temperature(thought) < 0.3 && age > 7days
  执行动作: 念头自动从显意识层迁移到潜意识层
  优先级: 低
```

#### 5.6.3 规则的编写方式

用户可以用三种方式编写规则:

| 方式 | 说明 | 适用人群 |
|---|---|---|
| **图形化配置** | 通过UI表单选择触发条件、判定逻辑、执行动作 | 所有用户 |
| **规则算法** | 用类自然语言或简单脚本编写规则 | 进阶用户 |
| **AI辅助编写** | 用户描述意图，AI生成规则算法 | 所有用户 |

#### 5.6.4 AI辅助编写规则

用户可以用自然语言描述想要的规则，AI生成对应的规则算法:

**交互流程**:
```
1. 用户打开"规则设置" → 点击"AI辅助"
2. 用户描述意图（自然语言）:
   "当两个念头有相同的标签，并且都在显意识层时，提示我合并"
3. AI生成规则算法:
   触发条件: tag(A) == tag(B) && layer(A) == layer(B) == "conscious"
   判定逻辑: sameTag && sameLayer
   执行动作: prompt("念头A和B标签相同且同层，是否合并？")
   优先级: 中
4. 用户预览规则（半透明预览，不直接生效）
5. 用户确认或调整
6. 确认后规则生效
```

**AI辅助的约束**（遵循L1-5）:
- ❌ AI不能直接执行规则（必须"半透明预览"，用户确认才生效）
- ❌ AI不能修改用户数据
- ✅ AI只能生成规则算法，由用户审核确认
- ✅ 规则生效前必须预览效果

#### 5.6.5 规则的优先级与冲突

当多条规则同时触发时，按优先级处理:

| 优先级 | 说明 |
|---|---|
| **高** | 互斥、包含等结构性规则 |
| **中** | 合并、并列等关系性规则 |
| **低** | 温度、年龄等时间性规则 |

**冲突处理**: 同优先级规则冲突时，提示用户选择，不自动执行。

#### 5.6.6 合并的具体流程

```
1. 用户拖动念头A → 拖到念头B上
2. 系统检查规则:
   - 是否有互斥规则？→ 是则斥力分离，终止
   - 是否有合并规则？→ 是则按规则处理
   - 无规则 → 默认提示"合并A和B？"
3. 念头B发亮提示（可合并）
4. 弹出提示:"合并A和B？"
   - 确认 → A和B合并为新实体
   - 取消 → A回到原位
5. 合并后的内容 = A的内容 + B的内容（按规则算法决定）
6. 合并后的层归属 = 按规则（默认取较深层）
7. 合并后形态 = 念头或记忆（按规则算法决定）
```

#### 5.6.7 拆分

合并的反操作，可将一个念头/记忆拆分为多个:

```
1. 长按念头 → 菜单 → 拆分
2. 选择拆分方式:
   - 按句子拆分（每句一个念头）
   - 按段落拆分（每段一个念头）
   - 手动选择拆分点
3. 确认 → 念头拆分为多个
4. 拆分后的念头继承原念头的关系（按规则）
```

### 5.7 标签数值与吸附系统

念头可以作为标签，标签可附加数值，通过算法决定念头在3D空间中的位置。

#### 5.7.1 念头作为标签

任何念头本身就是一个标签，可以被其他念头引用:

```
念头A "工作效率" → 本身是标签
念头B "写文档" → 引用标签"工作效率"
念头C "回复邮件" → 引用标签"工作效率"
```

#### 5.7.2 标签附加数值

标签不只是分类，还可以有数值:

```
标签"工作效率" + 数值0.8（重要度）
标签"家庭" + 数值0.9（重要度）
标签"学习" + 数值0.6（重要度）
```

数值可以是:
- 重要度（0-1）
- 优先级（整数）
- 自定义数值（用户定义含义）

#### 5.7.3 顺序吸附

多个标签按顺序吸附:

```
念头X 引用: [标签1, 标签2, 标签3]
→ 念头X吸附到 标签1→标签2→标签3 的路径上
→ 按顺序形成空间排列
```

#### 5.7.4 位置算法

用户可写算法，根据标签数值决定念头在3D空间的坐标:

```
算法示例1: 按重要度排列
  position.x = importance * 100
  position.y = 0
  position.z = 0

算法示例2: 多维度排列
  position.x = importance_value(tag1) * 100
  position.y = urgency_value(tag2) * 100
  position.z = frequency_value(tag3) * 100

算法示例3: 吸附到标签路径
  path = [tag1, tag2, tag3]
  position = interpolate_along_path(path, weight)
```

**空间坐标可读取**: 算法可以方便读取念头的当前坐标（x, y, z），用于计算或显示。

#### 5.7.5 算法的编写方式

与规则系统一致，支持三种方式:

| 方式 | 说明 |
|---|---|
| **图形化配置** | 通过UI表单选择标签、数值、排列方式 |
| **算法脚本** | 用类自然语言或简单脚本编写位置算法 |
| **AI辅助编写** | 用户描述意图，AI生成位置算法 |

### 5.8 念头引用系统

念头/记忆的正文里可以引用其他念头，引用格式为"念头名+(坐标)"，可选择是否显示。

#### 5.8.1 引用格式

```
正文内容... 引用念头"工作效率"（坐标: x=120, y=50, z=-30）... 继续正文
```

#### 5.8.2 显示控制

| 选项 | 说明 |
|---|---|
| **显示完整引用** | 念头名 + (坐标) 都显示 |
| **只显示念头名** | 隐藏坐标，只显示"工作效率" |
| **完全隐藏** | 引用不显示，但保持链接关系 |

#### 5.8.3 引用的交互

- 点击引用 → 跳转到被引用念头的位置（视角飞行）
- 长按引用 → 弹出被引用念头的预览（不跳转）
- 引用的念头被删除 → 引用标记为"已失效"，不自动删除

### 5.9 未知块标记系统

系统新增"未知"标记类型，用于标记用户尚未理解、尚未归类、尚未确定关系的念头/区块。

#### 5.9.1 未知标记的本质

| 属性 | 说明 |
|---|---|
| **本质** | 一种特殊的元标记，表示"用户对这块内容的认知状态" |
| **作用对象** | 念头、记忆、区块（封闭空间） |
| **与关系的关系** | 未知块不参与任何关系规则（合并/互斥/并列/包含/被包含），直到解除未知状态 |
| **与位置算法的关系** | 未知块不参与位置算法的坐标计算，悬浮在当前层但不受标签吸附 |
| **视觉表现** | 半透明+问号纹理+微弱漂浮动画，区别于已知念头 |

#### 5.9.2 手动标记为未知

| 操作 | 说明 |
|---|---|
| **念头级** | 念头菜单 → 标记为未知 |
| **区块级** | 选中区块 → 标记为未知（区块内所有念头继承未知状态） |
| **解除** | 再次点击未知标记 → 解除（可选择性保留子念头的未知状态） |

#### 5.9.3 规则自动标记为未知

用户可设定规则，让系统自动把符合条件的念头/区块标记为未知:

```
规则示例1: 关系缺失自动未知
  触发条件: 念头创建后 24 小时
  判定逻辑: relation_count(thought) == 0 && age > 24h
  执行动作: 标记为未知
  优先级: 低

规则示例2: 标签冲突自动未知
  触发条件: 念头被打上互斥标签
  判定逻辑: has_conflicting_tags(thought)
  执行动作: 标记为未知，提示用户整理解释
  优先级: 中

规则示例3: 内容过短自动未知
  触发条件: 念头文字 < 5 字
  判定逻辑: length(text) < 5
  执行动作: 标记为未知（可能是随手记的，未形成清晰想法）
  优先级: 低
```

#### 5.9.4 未知块的交互

| 操作 | 行为 |
|---|---|
| **点击未知块** | 弹出"这是什么？"输入框，引导用户写一句话解释 |
| **拖动未知块** | 自由移动，但不参与吸附/合并 |
| **长按未知块** | 菜单增加"转化为已知"选项 |
| **AI辅助** | AI可分析未知块内容，半透明预览建议标签/关系/归类（L1-5约束） |

#### 5.9.5 未知块的觉察意义

未知块是**自我觉察飞轮的加速器**——通过显式标记"我不知道这是什么"，把"模糊的不知道"变成"可见的待探索区域"，让用户在空间中直接看到自己的认知边界。

### 5.10 标记坐标规则算法

不止位置算法（§5.7）可写算法，**所有标记的坐标**都可设置规则算法。

#### 5.10.1 可设规则算法的标记坐标

| 标记类型 | 可设算法的坐标 | 示例 |
|---|---|---|
| **线标记** | 起点(x1,y1,z1) + 终点(x2,y2,z2) | "在两个互斥念头中点画一条分隔线" |
| **面标记** | 法向量 + 距原点距离 + 边界形状 | "在显意识层和潜意识层之间画分界面" |
| **体标记** | 中心坐标 + 尺寸 + 朝向 | "把温度>0.8的念头圈成一个体" |
| **阻尼组件** | 位置 + 朝向 + 阻尼强度 | "阻尼组件跟随最近7天未交互的念头群" |
| **参考线** | 位置 + 朝向 | "参考线沿X轴排列，间距按标签数值梯度" |

#### 5.10.2 标记坐标算法的结构

```
标记算法 = {
  触发条件: 何时重新计算标记坐标
  输入数据: 算法依赖的念头/标签/数值
  坐标计算: 根据输入数据计算标记的新坐标
  更新策略: 实时更新 / 防抖更新 / 手动触发
}
```

#### 5.10.3 标记坐标算法示例

```
算法示例1: 动态分界线
  触发条件: 念头位置变化时
  输入数据: 显意识层所有念头的Y坐标
  坐标计算:
    line.y = average(conscious_thoughts.y) - 10
    line.x = min_x to max_x
    line.z = min_z to max_z
  更新策略: 防抖更新（500ms）

算法示例2: 自适应阻尼组件位置
  触发条件: 念头温度变化
  输入数据: 温度<0.3的念头群
  坐标计算:
    damper.position = centroid(cold_thoughts)
    damper.orientation = align_to_dominant_axis(cold_thoughts)
  更新策略: 实时更新

算法示例3: 标签梯度参考线
  触发条件: 标签数值变化
  输入数据: 标签"重要度"的所有数值
  坐标计算:
    for each value in sorted(importance_values):
      line_x[i] = value * 100
      line_y[i] = 0
      line_z[i] = 0
  更新策略: 防抖更新
```

#### 5.10.4 算法的编写方式

与§5.6.3规则系统一致，支持三种方式:

| 方式 | 说明 |
|---|---|
| **图形化配置** | 通过UI表单选择触发条件、输入数据、坐标计算逻辑 |
| **算法脚本** | 用类自然语言或简单脚本编写标记坐标算法 |
| **AI辅助编写** | 用户描述意图，AI生成标记坐标算法（L1-5约束，半透明预览） |

### 5.11 记忆/笔记的内容格式

记忆和笔记的正文不是纯文本，是"行+块"的可组合结构，每个字/行/块都能附加多个维度。

#### 5.11.1 行制：一行/双行/三行

| 行制 | 说明 | 适用 |
|---|---|---|
| **一行**（默认） | 单行文字，未显式换行时默认 | 念头、短记录 |
| **双行** | 两行文字，可分别附加维度 | 对比、因果、问答 |
| **三行** | 三行文字，可分别附加维度 | 三段式、列表项 |

**关键约束**:
- 没有显式写成多行时，默认是一行
- 用户可主动切换为双行/三行
- 每行可独立附加标签、数值、坐标维度
- 每个字也可附加维度（如"重要字"加粗、"待澄清字"标黄）

#### 5.11.2 块制：一块/双块/三块

| 块制 | 说明 | 适用 |
|---|---|---|
| **一块** | 单个内容块 | 简单记忆 |
| **双块** | 两个块并排或上下 | 对比、引用+评注 |
| **三块** | 三个块 | 多角度、流程、列表 |

**块的本质**: 块是内容的基本单位，可以是:
- 文字块（一段文字）
- 念头块（嵌入一个念头作为块）
- 子记忆块（嵌入另一个记忆）
- 图片块
- 音频块
- 视频块
- 其他自定义块

#### 5.11.3 行内元素

行内可以填入多种元素，不只限于文字:

| 行内元素 | 说明 |
|---|---|
| **文字** | 默认行内元素 |
| **念头** | 嵌入一个念头作为行内元素（可点击进入） |
| **块引用** | 引用另一个块作为行内元素 |
| **图片** | 行内图片 |
| **音频** | 行内音频播放器 |
| **视频** | 行内视频 |
| **公式** | 行内公式 |
| **代码** | 行内代码 |
| **自定义** | 用户定义的其他行内元素类型 |

#### 5.11.4 块的连线

块之间可以连线，表达块级关系:

```
┌──────┐      ┌──────┐
│ 块A  │ ───→ │ 块B  │
│      │      │      │
└──────┘      └──────┘
      ↓
┌──────┐
│ 块C  │
└──────┘
```

| 连线类型 | 说明 |
|---|---|
| **顺序** | A→B，表示先后 |
| **因果** | A→B，表示因果 |
| **对比** | A↔B，表示对比 |
| **引用** | A→B，A引用B的内容 |
| **自定义** | 用户定义的连线类型 |

**连线与空间关系的关系**:
- 块连线是**正文内部**的关系
- §5.6的五种基础关系是**3D空间中**念头/记忆之间的关系
- 二者独立但可互转：用户可选择把块连线"提升"为空间关系

#### 5.11.5 文字成块

正常文字可以选择形成一个块:

```
正文: 这是一段普通文字，用户可以选择其中一部分
      ─────────────────────
      选中"普通文字" → 右键 → "形成块"
      → "普通文字"变成一个独立的行内块，可附加维度
```

**成块的意义**:
- 把"不重要的文字"变成"可标记、可连线、可附加维度的实体"
- 支持细粒度的内容组织
- 成块后的文字可独立被AI读取/编辑（L1-5约束）

#### 5.11.6 维度的叠加

每个字/行/块都能增加多个维度:

| 维度层级 | 可附加维度 |
|---|---|
| **字级** | 强调、待澄清、自定义标签、数值 |
| **行级** | 标签、数值、坐标偏移、连线 |
| **块级** | 标签、数值、坐标、关系、连线、自定义属性 |

**维度示例**:
```
块A（标签:工作, 数值:0.8）
  ├─ 行1（标签:紧急）
  │    ├─ 字"报"（强调）
  │    └─ 字"告"（待澄清）
  └─ 行2（标签:本周）
       └─ 文字内容...
```

#### 5.11.7 内容格式的简化原则

虽然行制+块制+行内元素+连线+维度叠加非常强大，但UI必须保持简洁:

1. **默认极简** — 不显式操作时，就是普通文字
2. **按需展开** — 用户需要时才切换为多行/多块
3. **视觉一致** — 行/块/连线的视觉风格与整体一致
4. **不破坏把玩感** — 内容编辑是浮层/展开态，不抢占基准面

### 5.12 复制系统

复制不被格式限制，保证可以按照格式复制到另一个笔记中。复制的信息使用转为代码的形式在剪贴板中转，防止系统不支持而丢失。

#### 5.12.1 复制的核心约束

| 约束 | 说明 |
|---|---|
| **不被格式限制** | 复制时保留所有结构信息（行制/块制/行内元素/连线/维度） |
| **可跨笔记粘贴** | 粘贴到另一个笔记时，按原格式恢复 |
| **代码中转** | 剪贴板中存储为代码形式，防止系统不支持而丢失 |
| **降级兼容** | 粘贴到不支持本格式的外部应用时，降级为纯文本 |

#### 5.12.2 剪贴板的代码中转

复制时，内容被序列化为**JSON格式**写入剪贴板:

```
剪贴板内容（示例）:

{
  "type": "nx-clip",
  "version": "1.0",
  "blocks": [
    {
      "id": "b1",
      "type": "text",
      "tags": ["工作"],
      "value": 0.8,
      "lines": [
        {
          "id": "l1",
          "tags": ["紧急"],
          "chars": [
            {"id": "c1", "text": "报"},
            {"id": "c2", "text": "告", "mark": "clarify"},
            {"id": "c3", "text": "内容"}
          ]
        },
        {
          "id": "l2",
          "tags": ["本周"],
          "chars": [{"text": "需要完成"}]
        }
      ]
    },
    {
      "id": "b2",
      "type": "thought",
      "ref": "thought-xyz123"
    }
  ],
  "connections": [
    {"from": "b1", "to": "b2", "type": "cause"}
  ]
}

# 同时写入纯文本降级版（供外部应用使用）
报告内容
需要完成
```

**双写策略**:
- **主写**: JSON格式（`nx-clip`），完整保留结构
- **副写**: 纯文本降级版，供外部应用使用
- **粘贴时优先读取**: 主写格式；读不到则读纯文本

#### 5.12.3 可复制的内容

| 内容 | 可复制 | 说明 |
|---|---|---|
| **文字** | ✅ | 纯文字 |
| **行** | ✅ | 含行级维度 |
| **块** | ✅ | 含块级维度+内部结构 |
| **念头** | ✅ | 作为块引用复制（保持引用关系） |
| **连线** | ✅ | 连同块一起复制 |
| **维度** | ✅ | 标签/数值/坐标等维度一并复制 |
| **图片/音频/视频** | ✅ | 作为块复制（引用资源） |
| **整篇记忆/笔记** | ✅ | 完整结构复制 |

#### 5.12.4 粘贴的行为

| 粘贴目标 | 行为 |
|---|---|
| **本应用内笔记** | 完整恢复原格式（行制/块制/连线/维度） |
| **本应用内念头** | 若内容复杂则提示"转为记忆？"，简单内容直接粘贴 |
| **外部应用（支持HTML）** | 降级为带基本格式的文本（粗体/列表等） |
| **外部应用（纯文本）** | 降级为纯文本，丢弃所有结构 |

#### 5.12.5 复制的引用处理

复制包含念头引用/块引用时:

| 情况 | 处理 |
|---|---|
| **同空间内粘贴** | 引用保持有效（指向原念头/块） |
| **跨空间粘贴** | 引用保持有效（通过ID定位） |
| **被引用念头已删除** | 引用标记为"已失效"，不自动删除 |
| **粘贴到外部** | 引用降级为"念头名+(坐标)"文本 |

#### 5.12.6 复制与导入导出的关系

| 维度 | 复制系统 | 导入导出（§15） |
|---|---|---|
| **范围** | 单个笔记/块/念头 | 所有用户自定义内容 |
| **中转** | 剪贴板（JSON格式） | 文件（JSON/Markdown） |
| **目的** | 笔记间快速搬运 | 跨设备/跨用户迁移 |
| **关系** | 复制系统是导入导出的"轻量级"版本，共享序列化格式 |

**共享序列化格式**: 复制系统的`nx-clip`格式是导入导出JSON格式的子集，保证一致性。

#### 5.12.7 AI与复制系统

AI可读取剪贴板中的JSON格式内容，进行编辑（L1-5约束）:
- AI读取复制的内容 → 分析结构
- AI生成粘贴建议（如"这个块应该连接到哪个念头"）
- 半透明预览建议，用户确认才生效

## §7 嵌套子空间

### 7.1 分形自相似

任意念头可进入，是子空间。子空间有完整的交互三粒度（消消乐/魔方/流形），分形自相似。

### 7.2 嵌套规则

| 规则 | 说明 |
|---|---|
| **进入** | 双指展开（念头上）→ 放大到进入 |
| **退出** | 双指捏合 → 缩小到退出 |
| **阻尼组件** | 子空间不默认放阻尼组件（但用户可选择） |
| **结构** | 子空间也有6层+2外置（或用户自定义框架） |

### 7.3 "魔方套魔方" = 流形的分形结构

```
父空间（3D晶体）
  ├─ 念头A → 进入 → 子空间A（3D晶体）
  │    ├─ 子念头A1 → 进入 → 子子空间A1
  │    └─ 子念头A2
  ├─ 念头B → 进入 → 子空间B
  └─ 念头C
```

## §9 Todo与笔记

### 9.1 Todo的多态性

Todo可以以多种形态出现，根据场景切换:

| 场景 | Todo形态 | 用途 |
|---|---|---|
| 快速查看 | 1/4操作区侧边栏 | 看一眼当前待办 |
| 拖入念头时 | 弹层 | 拖动念头到Todo区域，弹层展开 |
| 深度整理 | 特殊面 | 切换到Todo面，整面操作待办 |
| 复杂项目管理 | 子晶体 | 进入Todo子空间，3D整理 |

### 9.2 念头/笔记的二元性

| 区域 | 可创建 | 默认归属 |
|---|---|---|
| **念头区** | 念头、笔记 | 念头进入3D空间，笔记可选择放入或不放入 |
| **Todo区** | 念头、笔记 | 默认不进入3D空间 |

### 9.3 笔记的特性

| 特性 | 说明 |
|---|---|
| **临时记录** | 快速记下，不进入3D空间 |
| **悬而未决** | 还没决定是念头还是行动 |
| **可转化** | 之后可以转为念头放入3D空间 |

### 9.4 双向创建

- 念头区创建的念头 → 默认进入3D空间
- 念头区创建的笔记 → 可以放入3D空间，也可以不放入
- Todo区创建的念头 → 可以拖入3D空间
- Todo区创建的笔记 → 默认不放入3D空间，可临时记录或悬而未决

## §10 辅助功能（整合到统一界面）

**核心原则**: 撤销/重做、多选、搜索、设置入口这些功能是必要的，但不能破坏界面的把玩感和统一性。

| **入口** | 1/4操作区的3D缩略图旁边有小图标 |

| **操作** | 底部出现统一操作栏（合并/移动/删除/加入Todo） |

### 10.3 搜索

| 方式 | 说明 |
|---|---|
| **入口** | 1/4操作区顶部有搜索图标 |
| **呈现** | 搜索结果在基准面上高亮，不破坏空间结构 |
| **约束** | 搜索是过滤，不是跳转——念头的空间位置不变 |

### 10.4 设置入口

| 方式 | 说明 |
|---|---|
| **入口** | 1/4操作区的设置图标 |
| **内容** | 认知框架切换、层定义、轨道自定义、阻尼组件设置等 |
| **约束** | 设置面板是浮层，不破坏基准面的把玩感 |

### 10.5 界面统一性原则

所有辅助功能遵循:
1. **不抢占基准面** — 辅助功能在1/4操作区或浮层
2. **不破坏把玩感** — 手势简洁，不增加认知负担
3. **视觉一致** — 颜色/字体/动画与主体一致
4. **可隐藏** — 辅助功能可关闭，纯净把玩

## §17 性能边界与认知负荷

### 17.1 性能与认知负荷匹配

性能边界与用户的认知负荷相匹配——不是技术能支持多少，而是用户能管理多少。

### 17.2 数量限制

| 维度 | 限制 | 超出处理 |
|---|---|---|
| **单个3D空间念头数** | 用户可限定数量（默认如100-200） | 超过上限提示用户整理念头 |
| **嵌套深度** | **不限制** | 靠用户自己管理深度 |
| **子空间数量** | 不限制 | 靠用户自己管理 |

### 17.3 超过上限的处理

当念头数量超过用户设定的上限时:
1. **提示用户** "当前空间已有N个念头，建议整理念头"
2. **建议整理方式**:
   - 用"包含"关系把相关念头打包成子念头
   - 合并相似念头
   - 把沉寂念头移到潜意识层
3. **不强制阻止** 用户可以忽略提示继续创建

### 17.4 用念头包含解决数量问题

用户可以用"包含"关系把多个念头打包成一个父念头:
```
100个零散念头 → 包含关系 → 10个父念头（每个包含10个子念头）
→ 父念头层面只看到10个，子念头进入子空间
```

这是"用念头包含的方式解决性能边界"的核心思路——分形嵌套既是交互模式，也是性能管理方式。

### 17.5 移动端性能基准

| 指标 | 基准 |
|---|---|
| **帧率** | 60fps（念头数在上限内） |
| **响应时间** | 手势响应<16ms |
| **加载时间** | 空间加载<1秒（100念头） |

## §19 完整使用路径验证

### 19.1 第一次使用路径

```
1. 用户打开软件 → 看到有限3D晶体（空），正面基准面
2. 长按空白 → 弹出输入框 → 写"今天很累" → 念头出现在显意识表层

8. 右划1 → 切到念头区/Todo视角

12. 双指捏合 → 退出子空间
```

4. **新觉察笔记**: 用户写"关于觉察"的念头 → 飞轮回到1
5. **飞轮持续**: 新笔记融入拓扑 → 空间结构更新
