# 拓扑意识空间 — 重做设计 Spec

> 对 Phase 0 原型的**根本性重做**。旧原型是"连线+2D 星团"，新设计是"不规则多面体拓扑意识空间"。
> 这不是修补，是重新定义产品可视化的核心范式。

| 项目 | 值 |
|---|---|
| 创建日期 | 2026-07-05 |
| 优先级 | **P0 重做**(取代旧的 phase-0-lightweight-restructure spec) |
| 周期 | 5-7 天 |
| 前置依赖 | geb-infrastructure-bootstrap(PR #2, #3 已就位) |

## §1 核心洞察(从用户原话提取)

> "连线不显式，用多种几何的意识空间坐标系来定位"
> "立体式不规则多面体模仿潜意识"
> "模拟真实的拓扑体为意识空间"

### 1.1 本质
- **旧范式**：节点+连线 = 知识图谱(机械、显式、理性)
- **新范式**：念头 = 空间中的位置关系 = 拓扑流形(有机、隐含、潜意识的)
- **连线消失**：两个念头的"关系"由它们在空间中的**相对位置、曲率、密度**来暗示，而不是画一条线
- **多面体**：一组念头形成的不规则凸包 = 一个"思考簇"，其表面就是该思考簇的"意识边界"

### 1.2 一句话定位
**把"念头田野"从 2D 节点连线图升级为 3D 不规则多面体拓扑空间——念头是空间中的点，思考簇是凸包多面体，关系是曲率与距离。**

## §2 技术选型

### 2.1 技术栈替换

| 旧 | 新 | 理由 |
|---|---|---|
| PixiJS 2D | **Three.js** WebGL 3D | 3D 不规则多面体必需 |
| d3-force 2D | **d3-force-3d** | z 轴受力 |
| 5 种关系类型(线样式) | **空间位置暗示**(距离/角度/密度) | 连线不显式 |
| G2 星团布局 | **凸包多面体 + Voronoi 区域** | 拓扑体 |
| Yjs + IndexedDB | **不变**(持久化层) | 数据模型不变 |
| Vitest | **不变** | 测试框架不变 |

### 2.2 新增核心依赖

| 包 | 用途 |
|---|---|
| `three` | WebGL 3D 引擎 |
| `d3-force-3d` | 3D 力导向仿真 |
| `3d-force-graph` 或自研 | 力导向图封装(可选,可能自研更灵活) |
| `convex-hull` (npm: `convex-hull`) | 3D 凸包计算(念头→多面体) |

## §3 视觉设计(概念层)

### 3.1 基本外观
- **底色**：极深灰黑(潜意识深渊)，微弱的径向渐亮(意识中心)
- **念头节点**：半透明光球，大小随 mass 变化，颜色随 temperature 从暖(金)到冷(深蓝)
- **思考簇(多面体)**：半透明不规则凸包，顶点是念头位置，表面有微弱的晶格线(暗示结构但不喧宾夺主)
- **空间本身**：空不是空的——微弱的星尘粒子散布整个空间，暗示"思维的背景噪声"

### 3.2 三轴意识坐标系(核心创新)

意识空间的三个物理轴各对应一个意识维度：

| 轴 | 方向 | 意识含义 | 视觉表现 |
|---|---|---|---|
| **Y 轴** | 上→下 | **显意识 → 潜意识深度** | 上方亮(显意识层)，下方暗(潜意识深渊)。念头越往下沉，越"深埋" |
| **Z 轴** | 前(近相机)→后(远) | **意识出现频率** | 前面=高频反复出现的念头(温暖明亮)，后面=低频偶尔闪现的念头(黯淡稀薄) |
| **X 轴** | 左→右 | **概念关联的横向扩散** | 关联度高的念头沿X轴自然聚拢成簇 |

**Y 轴分层(关键)**
- **L0 显意识层**(Y > 200)：正在活跃思考的念头——温度高，明亮
- **L1 近潜意识层**(0 ≤ Y ≤ 200)：半活跃的念头——半透明，缓慢漂移
- **L2 潜意识层**(-200 ≤ Y < 0)：沉睡的念头——暗沉，缓慢下沉
- **L3 深层潜意识**(Y < -200)：被遗忘的念头——几乎不可见，沉积在底部

### 3.3 魔方式交互模型(核心创新)

**不是**自由轨道旋转。**是**像观察一颗魔方一样，从**6 个预定义的面**查看意识空间。

| 面 | 相机位置 | 能看到什么 |
|---|---|---|
| **正面(Front)** | Z+ → 看 XY 平面 | 显意识→潜意识分层(Y轴)+ 概念扩散(X轴)，完整的"意识剖面" |
| **背面(Back)** | Z- → 看 XY 平面(反向) | 低频念头的视角——那些偶尔闪现的、被遗忘的 |
| **顶面(Top)** | Y+ → 看 XZ 平面 | 鸟瞰——俯瞰整个意识拓扑，看到所有念头的空间分布 |
| **底面(Bottom)** | Y- → 看 XZ 平面(仰视) | 潜意识深渊视角——从底部向上看沉积的念头 |
| **左面(Left)** | X- → 看 YZ 平面 | 从"概念关联"的左侧看纵向的意识分层 |
| **右面(Right)** | X+ → 看 YZ 平面 | 从"概念关联"的右侧看，可能是不同领域的念头簇 |

**切换方式**：
- **触屏滑动**：快速滑动(Swipe)切换到相邻面。水平滑动=绕Y轴旋转到下一个面，垂直滑动=绕X轴旋转到下一个面
- **吸附动画**：切换时有 0.4秒的缓动动画，相机平滑飞抵目标面的位置
- **面指示器**：屏幕角落显示当前正在看哪个面(小魔方示意图，当前面高亮)
- **手势灵敏度**：滑动超过 30% 屏幕宽度=触发面切换；不到=回弹

### 3.4 潜意识 = 不规则多面体拓扑

- **没有完美的正多面体** — 每个多面体都是不规则凸包
- 多面体的**每个面微微扭曲**(在顶点上加 Perlin 噪声偏移)
- 拓扑体的**表面硬度**不同 — 密集区"硬"(半透明更低)，稀疏区"软"(更透)
- 多面体之间可以有**引力/斥力** — 它们像天体一样互相绕行
- **Y轴位置决定显隐程度**——高处的多面体更亮更实，低处的更暗更透

### 3.5 连线不显式，但关系仍在

- 两个念头的"相关度" = 它们在 3D 空间中的**测地距离**(在某个曲率场中)
- 不是欧氏距离，而是**被拓扑体曲面弯曲后的距离**
- 相关度高的念头**自然靠近**(力导向 + 自定义力：引力 ∝ 关系强度)
- 用户不需要"看到连线"——空间布局本身暗示了关系
- **选中一个念头时**：相关的念头发微光(半透明高亮)，持续 2 秒，然后渐隐——这是唯一"显式"的关系提示，其他时候纯靠空间位置

## §4 交互设计

### 4.1 魔方面切换(核心交互)
| 操作 | 行为 |
|---|---|
| **水平滑动**(左右) | 绕 Y 轴旋转到相邻面。正面→右面→背面→左面→正面(四向循环) |
| **垂直滑动**(上下) | 绕 X 轴旋转到相邻面。正面→顶面→背面→底面→正面(四向循环) |
| **双指缩放** | 拉近/拉远视角(zoom in/out) |
| **双指旋转** | 在当前面的基础上微调视角(±15° 的自由度,超过即吸附回最近面) |
| **单击空白** | 在当前相机前方投下一颗新念头(Y 坐标：相机俯视角度越大，投下的念头 Y 越高=越"显意识") |

### 4.2 念头操作
| 操作 | 行为 |
|---|---|
| **单击念头** | 选中，念头发光。相关念头发微光 2 秒(关系暗示) |
| **双击念头** | 打开详情面板——编辑 text，删除，标记"忽略" |
| **拖拽念头** | 在空间内移动位置(力导向暂时 override) |
| **Shift+单击** | 多选离散念头 |
| **Ctrl+框选** | 框选一组念头(在 2D 屏幕上画矩形,投影到 3D) |

### 4.3 提炼(Crystallize)
- 选中一组念头 → 点击工具栏"提炼" → 弹出几何类型选择(Hull/Voronoi/Spiral/Star/Grid/Free)
- 选择后 → 0.8 秒动画:念头收缩到凸包顶点→多面体面片渐出→粒子飘散
- 提炼结果=一个新的"拓扑体节点"，可继续参与力导向

### 4.4 沉积层

- **自动下沉**:温度<0.3 的念头开始向 Y 轴负方向缓慢漂移(速度 2px/帧)
- **捞起操作**:双击沉积层的念头→温度恢复为 1→浮回显意识层(Y>200)
- **视觉**:沉积层的念头半透明、密度降低、呈暗蓝色

### 4.5 面指示器

- 屏幕右下角显示一个小型半透明魔方(边长 60px)
- 当前正在看的面高亮(发光边框)
- 相邻面半透明
- 用户滑动时魔方跟着旋转

## §5 数据结构(不变，但渲染层完全重写)

### 5.1 核心数据(Yjs schema)
- `thoughts`: Y.Map — 每个念头有 `id, text, x, y, z, mass, temperature, lastInteractionAt`
- `edges`: Y.Map — 边(仍然存在，但**不渲染**，只用于力导向计算和结晶判定)
- `structures`: Y.Map — 拓扑体(多面体)的元数据(顶点列表、几何类型、凸包面片)

### 5.2 Edge 的新角色
- edge 不再被可视化
- edge 仍然存在于数据中，用于：
  - 力导向中的引力计算(有 edge 的节点之间额外引力)
  - 内聚度评分(结晶判定)
  - 未来 Phase 2 的语义搜索权重

## §7 验收标准

- [ ] 打开页面看到一个 3D 深色空间(不是 2D 平面)
- [ ] 单击投下念头后，它在 3D 中漂浮
- [ ] 鼠标可以旋转/缩放/平移 3D 视角
- [ ] 选中一组念头 → 可以"提炼"成一个不规则凸包多面体
- [ ] 没有连线(edge 不可见)
- [ ] 念头之间通过空间位置暗示关系(有 edge 的念头在力导向下自动靠近)
- [ ] 6 种几何布局至少 3 种可用(凸包+Voronoi+螺旋优先)
- [ ] 低温念头自动下沉到沉积层
- [ ] 体感"顺滑" ≥ 7/10(用户主观)

## §8 风险

| 风险 | 缓解 |
|---|---|
| Three.js 学习曲线 | 已有大量社区模板，优先复刻而非自研 |
| 凸包计算性能(50+ 节点) | convex-hull 用 incremental 算法，O(n log n)，50 节点毫秒级 |
| 3D 交互比 2D 难用 | 默认视角 45° 俯角 + 锁定 z 轴拖拽(可选) |
| 用户觉得"看不到连线迷失方向" | 提供"高亮相关念头"的功能(选中一个后，相关念头发光) |
