# 拓扑意识空间 — 重做设计 Spec

> 对 Phase 0 原型的**根本性重做**。旧原型是"连线+2D 星团"，新设计是"不规则多面体拓扑意识空间"。
> 这不是修补，是重新定义产品可视化的核心范式。

| 项目 | 值 |
|---|---|
| 创建日期 | 2026-07-05 |
| 优先级 | **P0 重做**(取代旧的 phase-0-lightweight-restructure spec) |
| 周期 | 5-7 天 |
| 前置依赖 | geb-infrastructure-bootstrap(PR #2, #3 已就位) |

---

## §1 核心洞察(从用户原话提取)

> "连线不显式，用多种几何的意识空间坐标系来定位"
> "立体式不规则多面体模仿潜意识"
> "模拟真实的拓扑体为意识空间"

### 1.1 本质
- **旧范式**：节点+连线 = 知识图谱(机械、显式、理性)
- **新范式**：念头 = 空间中的位置关系 = 拓扑流形(有机、隐含、潜意识的)
- **连线消失**：两个念头的"关系"由它们在空间中的**相对位置、曲率、密度**来暗示，而不是画一条线
- **多面体**：一组念头形成的不规则凸包 = 一个"思考簇"，其表面就是该思考簇的"意识边界"

### 1.2 一句话定位
**把"念头田野"从 2D 节点连线图升级为 3D 不规则多面体拓扑空间——念头是空间中的点，思考簇是凸包多面体，关系是曲率与距离。**

## §2 技术选型

### 2.1 技术栈替换

| 旧 | 新 | 理由 |
|---|---|---|
| PixiJS 2D | **Three.js** WebGL 3D | 3D 不规则多面体必需 |
| d3-force 2D | **d3-force-3d** | z 轴受力 |
| 5 种关系类型(线样式) | **空间位置暗示**(距离/角度/密度) | 连线不显式 |
| G2 星团布局 | **凸包多面体 + Voronoi 区域** | 拓扑体 |
| Yjs + IndexedDB | **不变**(持久化层) | 数据模型不变 |
| Vitest | **不变** | 测试框架不变 |

### 2.2 新增核心依赖

| 包 | 用途 |
|---|---|
| `three` | WebGL 3D 引擎 |
| `d3-force-3d` | 3D 力导向仿真 |
| `3d-force-graph` 或自研 | 力导向图封装(可选,可能自研更灵活) |
| `convex-hull` (npm: `convex-hull`) | 3D 凸包计算(念头→多面体) |

## §3 视觉设计(概念层)

### 3.1 基本外观
- **底色**：极深灰黑(潜意识深渊)，微弱的径向渐亮(意识中心)
- **念头节点**：半透明光球，大小随 mass 变化，颜色随 temperature 从暖(金)到冷(深蓝)
- **思考簇(多面体)**：半透明不规则凸包，顶点是念头位置，表面有微弱的晶格线(暗示结构但不喧宾夺主)
- **空间本身**：空不是空的——微弱的星尘粒子散布整个空间，暗示"思维的背景噪声"
- **相机**：默认从 45° 俯角观察，用户用鼠标旋转/缩放/平移

### 3.2 多种几何坐标系(对应 "多种几何的意识空间坐标系")

每个"思考簇"可以有**不同的内部几何布局**：

| 几何类型 | 视觉 | 适合场景 |
|---|---|---|
| **凸包(Hull)** | 念头在顶点，多面体包裹 | 一组互相强关联的念头 |
| **Voronoi 区域** | 空间被念头"瓜分"，每个念头有一个不规则领地 | 互斥的概念(矛盾、对比) |
| **螺旋(Spiral)** | 念头沿 3D 螺旋线排列 | 时序递进的思考链 |
| **星爆(Star)** | 中心一个核念头，周围辐射散开 | 一个核心概念的多方向展开 |
| **网格(Grid)** | 念头在 3D 规则网格上 | 系统性的分类 |
| **自由漂浮(Free)** | 念头在 3D 空间中散布，无显式几何 | 未成形的朦胧念头 |

用户对一组念头执行"提炼"操作时，可以选择以上任一种几何。

### 3.3 潜意识 = 不规则性
- **没有完美的正多面体** — 每个多面体都是不规则凸包
- 多面体的**每个面微微扭曲**(在顶点上加 Perlin 噪声偏移)
- 拓扑体的**表面硬度**不同 — 密集区"硬"(半透明更低)，稀疏区"软"(更透)
- 多面体之间可以有**引力/斥力** — 它们像天体一样互相绕行

### 3.4 连线不显式，但关系仍在
- 两个念头的"相关度" = 它们在 3D 空间中的**测地距离**(在某个曲率场中)
- 不是欧氏距离，而是**被拓扑体曲面弯曲后的距离**
- 相关度高的念头**自然靠近**(力导向 + 自定义力：引力 ∝ 关系强度)
- 用户不需要"看到连线"——空间布局本身暗示了关系

## §4 交互设计

### 4.1 核心操作
| 操作 | 行为 |
|---|---|
| 单击空白 | 在当前相机前投下一颗新念头 |
| 拖拽念头 | 手动调整位置(暂时 override 力导向) |
| 双击念头 | 打开详情(不变)——编辑 text，删除 |
| 右键拖拽 | 旋转 3D 视角 |
| 滚轮 | 缩放 |
| Ctrl+左键框选 | 选中一组念头 |
| Ctrl+Shift+单击 | 多选念头(离散) |
| 选中一组后 → 右键菜单 | "提炼为凸包" / "Voronoi 分区" / "螺旋排列" |

### 4.2 提炼(Crystallize)重构
- **旧**：手动触发 + 内聚度评分 → 粒子动画
- **新**：选中一组念头 → 选择几何类型 → 3D 凸包动画(顶点收缩到位置→多面体面片渐出→粒子从表面飘散)
- 提炼结果 = 一个新的"拓扑体节点"(一个多面体，可以继续参与更大的空间布局)

### 4.3 撤销重做(保留)
- Ctrl+Z / Ctrl+Y，基于 Yjs UndoManager

### 4.4 温度衰减(保留，视觉升级)
- 温度低 → 节点变暗变小，慢慢沉到空间底部的"沉积层"
- 沉积层 = 空间的底部区域，半透明的念头沉在那里，可以随时"捞起来"重新激活

## §5 数据结构(不变，但渲染层完全重写)

### 5.1 核心数据(Yjs schema)
- `thoughts`: Y.Map — 每个念头有 `id, text, x, y, z, mass, temperature, lastInteractionAt`
- `edges`: Y.Map — 边(仍然存在，但**不渲染**，只用于力导向计算和结晶判定)
- `structures`: Y.Map — 拓扑体(多面体)的元数据(顶点列表、几何类型、凸包面片)

### 5.2 Edge 的新角色
- edge 不再被可视化
- edge 仍然存在于数据中，用于：
  - 力导向中的引力计算(有 edge 的节点之间额外引力)
  - 内聚度评分(结晶判定)
  - 未来 Phase 2 的语义搜索权重

## §7 验收标准

- [ ] 打开页面看到一个 3D 深色空间(不是 2D 平面)
- [ ] 单击投下念头后，它在 3D 中漂浮
- [ ] 鼠标可以旋转/缩放/平移 3D 视角
- [ ] 选中一组念头 → 可以"提炼"成一个不规则凸包多面体
- [ ] 没有连线(edge 不可见)
- [ ] 念头之间通过空间位置暗示关系(有 edge 的念头在力导向下自动靠近)
- [ ] 6 种几何布局至少 3 种可用(凸包+Voronoi+螺旋优先)
- [ ] 低温念头自动下沉到沉积层
- [ ] 体感"顺滑" ≥ 7/10(用户主观)

## §8 风险

| 风险 | 缓解 |
|---|---|
| Three.js 学习曲线 | 已有大量社区模板，优先复刻而非自研 |
| 凸包计算性能(50+ 节点) | convex-hull 用 incremental 算法，O(n log n)，50 节点毫秒级 |
| 3D 交互比 2D 难用 | 默认视角 45° 俯角 + 锁定 z 轴拖拽(可选) |
| 用户觉得"看不到连线迷失方向" | 提供"高亮相关念头"的功能(选中一个后，相关念头发光) |
