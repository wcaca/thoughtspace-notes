# S2.12 ExpectedFrameCalculator — 理论帧耗时 + overhead 三档告警

> 状态: 已实做 (2026-07-13)
> 配套: `src/v2/render/expected-calculator.js` + `tests/v2/expected-calculator.test.js`
> 集成: `src/v2/render/render-pipeline.js` (getStats 加 4 字段) + `src/v2/debug/debug-overlay.js` (panel 加 2 行)

## WHY (为什么)

S2.10 暴露了 `recentAvgMs` (实测), 但**实测 ms 受 GC / raf jitter / V8 优化影响**, 不能直接当 baseline。

用户问"我还能不能加一个念头?" → 看 `remainingBudget`, 不用肉眼看 ms 数
系统报警"为什么卡了?" → 看 `overhead`, 不用对比两次实测

**目标**: 单独算"设计应该多少", 超 = 有问题 (缓存未命中 / 频繁分配 / 误触 render)

## 经验常数 (来自 S1/S2 真实测量, 单位 ms)

| 阶段 | 常数 | 来源 |
|---|---|---|
| input | 0.3 | event 收集 + 时间戳同步 (S1) |
| state | 1.0 | stateStore diff + 触发器匹配 (S1) |
| transform | 1.5 | matrix 批量更新 + Frustum culling (S1) |
| render | 4.0 | three.js draw call 排队 + GPU 上传 (S1) |
| snapshot | 0.8 | Y.encode 增量序列化 (S2.6 thought-bridge) |
| base | 0.5 | raf 调度 + ctx 构造 + stats 收集 |

**总和 8.1ms** (远小于 16ms 预算, 留 7.9ms 给用户逻辑)

## 接口

```js
import {
  DEFAULT_EXPECTED_CONSTANTS,
  computeExpectedMs,
  computeOverhead,
  computeRemainingBudget,
} from './expected-calculator.js';

// 理论帧耗时
const { expectedMs, byStage, callbackMultiplier } = computeExpectedMs({ stages: pipeline._stages });
// 实际 vs 理论
const { overheadMs, overheadPct, severity } = computeOverhead(actualMs, expectedMs);
// 剩余预算 (还能加多少回调)
const { remainingMs, byStage, note } = computeRemainingBudget(stages);
```

## 关键决策

1. **常数从测量取, 不空想** (调整时机: 加新模块 e.g. S3 physics 再回测, 别提前改)
2. **overhead 阈值**: >50% alarm / >20% warn / else ok (经验值, 可调)
3. **callbackMultiplier**: 解释"加个回调预期涨多少" (例: 加 1 个 transform, multiplier = 1.5/sum ≈ 18.5%)
4. **computeRemainingBudget 16ms 默认** = 60fps 预算 (易改 30fps=33 / 120fps=8)
5. **render-pipeline getStats 加 4 字段** (后向兼容: 旧 stats consumer 不读也不报错)

## 集成

### render-pipeline getStats()
新增 4 字段: `expectedMs` / `overheadMs` / `overheadPct` / `severity`
其余字段保持不变, S2.10 测试无需改

### debug-overlay panel
- 总览段加 2 行: `expected` / `overhead (X%)`
- overhead 行: `severity` 染色 (alarm=红, warn=琥珀, ok=白)

## 性能

- `computeExpectedMs` 是 O(5) 循环 + 一次 reduce, < 0.01ms
- 5Hz 采样频率 + 简单算术 = 0.001% frame budget
- 不会拖慢 S2.11 overlay 自身 (S2.11 plan 担心 < 0.5ms, 实测 < 0.1ms)
