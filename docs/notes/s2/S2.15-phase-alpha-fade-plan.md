# S2.15 Phase Alpha Fade — 相变 alpha 渐入

> 状态: 计划 (2026-07-18)
> 配套: `src/v2/render/thought-mesh.js` (_computePhaseAlphaMod + onBeforeCompile shader patch) + `tests/v2/phase-alpha-fade.test.js` (新, 13 测试)
> 集成: 跟 S2.14 scale 弹出协同 (一个动 scale, 一个动 alpha), 形成"念头从无到有"双轴动画
> 上游: S2.14 phase-transition 入场动画 (scale lerp) + S2.8 Thought.phase + S2.10 render-pipeline (tickPhaseTransitions)

## WHY (为什么)

S2.14 入场动画只动 scale (0→1 弹入), 视觉是"突然出现". 用户反馈: "念头凭空弹出, 缺少'从无到有'的渐变感".

**S2.15 = 视觉补完**:
- **scale 弹出** (S2.14) 跟 **alpha 渐入** (S2.15) 同步, scale=0 时 alpha≈0.4, scale=1 时 alpha=1.0
- **shader 走 instanced attribute** (`aPhaseProgress`), CPU 算 alpha mod 写到 instance, GPU 端 fragment shader 乘到 gl_FragColor.a
- **onBeforeCompile hook** 注入 vertex+fragment, 不替换整个 shader, 保持 MeshStandardMaterial 其他特性 (PBR/光照/温度色)

## 现状 (问题点)

```js
// S2.14 thought-mesh.js:_computePhaseScaleMod
const phaseScaleMod = this._computePhaseScaleMod(thought);
const finalScale = displayScale * phaseScaleMod;
// scale 0→1 弹出
```

但 material 是 `transparent: true` + `opacity: 0.85` 静态. SEED phase progress=0 时 mesh 仍以 0.85 alpha 渲染, 用户看到"小而亮的方块"凭空出现, 不是"渐渐浮现".

## 修复方案

### 1. _computePhaseAlphaMod helper (纯函数, 易测)

```js
_computePhaseAlphaMod(thought) {
  const currentPhase = thought._transient?.currentPhase ?? ThoughtPhase.SEED;
  const phaseProg = thought._transient?.phaseTransitionProgress ?? 0;
  if (currentPhase === ThoughtPhase.SEED) {
    // SEED 起步: alpha=0, 进度→1 时 alpha→1.0 (淡入)
    return phaseProg;  // 跟 scale 同步
  }
  if (phaseProg < 1) {
    // 其他相变中: 0.4 → 1.0 变实感 (alpha 起始 0.4 < scale 起始 0.7, 视觉更"先淡后实")
    return 0.4 + 0.6 * phaseProg;
  }
  return 1.0;
}
```

### 2. onBeforeCompile shader patch (核心)

```js
this.material.onBeforeCompile = (shader) => {
  // 1. vertex shader 注入: attribute + varying
  shader.vertexShader = 'attribute float aPhaseProgress;\n' + shader.vertexShader;
  shader.vertexShader = shader.vertexShader.replace(
    '#include <common>',
    '#include <common>\nvarying float vAlphaMod;'
  );
  shader.vertexShader = shader.vertexShader.replace(
    '#include <begin_vertex>',
    '#include <begin_vertex>\nvAlphaMod = 0.4 + 0.6 * aPhaseProgress;'
  );
  // 2. fragment shader 注入: varying + alpha 调整
  shader.fragmentShader = shader.fragmentShader.replace(
    '#include <common>',
    '#include <common>\nvarying float vAlphaMod;'
  );
  shader.fragmentShader = shader.fragmentShader.replace(
    '#include <output_fragment>',
    '#include <output_fragment>\ngl_FragColor.a *= vAlphaMod;'
  );
};
```

### 3. InstancedBufferAttribute 已存在 (S2.14 阶段已 setAttribute)

```js
// S2.14 已加: this.geometry.setAttribute('aPhaseProgress', new THREE.InstancedBufferAttribute(this._phaseProgressArr, 1));
// S2.15 重用, 不需新增 attribute
```
