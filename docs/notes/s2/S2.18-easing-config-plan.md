# S2.18 缓动曲线可配置 (Easing Curves Configurable)

## WHY

- S2.16 只实现了 `easeOutCubic`, 写死在 `_applyPhaseEasing`
- 不同场景需要不同缓动:
  - **ease-in** (前慢后快): 退场动画 (fadeOut/scaleOut) "加速离开"
  - **ease-in-out** (前后慢中间快): 循环动画 (pulse/breathe) "自然进出"
  - **linear** (匀速): 调试/对比
- S2.16 plan 留 "S2.18 缓动曲线可配置" 候选
- 用户需要 phase-transition 之外的动画也能选缓动

## WHAT

**2 改**:

1. `src/v2/animation/ease.js`:
   - 新 `easeInCubic(x)` — 公式 `y = x^2.5` (跟 easeOut 对称, easeIn(0.5) + easeOut(0.5) = 1)
   - 新 `easeInOutCubic(x)` — 前半 easeIn + 后半 easeOut, 折点 0.5 = 0.5
   - `EasingType` 加 `EASE_IN` / `EASE_IN_OUT`, 共 4 种
   - `applyEasing` 真正按 type 分发 (之前只识 LINEAR + 默认 ease-out)
   - 未识 type fallback 到 ease-out (防 typo 崩溃, 跟 S2.16 兼容)

2. `tests/v2/phase-easing.test.js`:
   - 新 22 测试 (easeInCubic 5 + easeInOutCubic 7 + applyEasing 分发 8 + 类型枚举 2)
   - 关键验证: easeIn + easeOut 互补 = 1, easeInOut 对称性, applyEasing 4 种分发

## 4 种缓动语义

| type | 行为 | 场景 |
|---|---|---|
| `linear` | 匀速, 无缓动 | 调试/对比 |
| `ease-out` | 前快后慢 (默认) | phase-transition (S2.16 沿用) |
| `ease-in` | 前慢后快 | 退场动画 (fadeOut/scaleOut) |
| `ease-in-out` | 前后慢中间快 | 循环动画 (pulse/breathe) |

## 经验

- **3 条曲线数学对称**: easeOut(x) + easeIn(x) = 1 (严格), easeInOut(0.25) + easeInOut(0.75) = 1
  - 测试用对称性验证 3 条曲线实现正确
- **简化版公式够用**: `y = x^2.5` (easeIn) / `y = 1 - (1-x)^2.5` (easeOut) 视觉跟 CSS cubic-bezier 差 < 5%
  - 不需要 Newton-Raphson 解严格 bezier
- **applyEasing fallback 重要**: 末识 type 走 ease-out 跟 S2.16 兼容, 防止未来 typo 崩溃
- **EasingType frozen 不可变**: 用 `Object.freeze`, 测试 `Object.isFrozen` 验
