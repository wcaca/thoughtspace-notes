# S2.13 QuickAddPanel — 用户可见的念头快速录入 UI

> 状态: 计划 (2026-07-15)
> 配套: `src/v2/interaction/quick-add-panel.js` (待实做) + `tests/v2/quick-add-panel.test.js` (待实做)
> 集成: `src/v2/main.js` + `src/v2/core/action-router.js` (dispatch ADD_THOUGHT)
> 上游: S2.8 thought 实体 + S2.10 render-pipeline + S2.11 DebugOverlay

## WHY (为什么)

S2.8 实做了 Thought 实体, S2.10 暴露了 `__v2.spawnSampleThought()` 调试入口,
但**用户看不到任何 UI** — 所有 Thought 都是 main.js 启动时塞的 3 个示例。
AI 排查走 `__v2.spawnSampleThought()` 但用户没有"添加自己念头"的入口。

7-14 backlog 写 "S2.13 计划 (拓扑阶段入场动画 / interaction zone)" —
S2.13 选 **quick-add UI** (interaction zone 之一), 是用户首次与 v2 系统交互的入口。

**目标**: 用户点一个按钮 → 浮层 → 输文字 → 选 layer → 点确认 → Thought 实化到 3D 空间

## 约束 (来自 design review + S1 quick-add 经验)

1. **首屏可见**: 启动后主视图右上角有 `+ 念头` 按钮, 任何用户都能找到
2. **3 步极简**: 点按钮 → 输文字 → 选层, 不超过 3 步完成
3. **键盘可达**: `N` 键 = 打开 quick-add (跟 notion / obsidian 习惯一致)
4. **浮层开销 < 0.3ms** (DOM 创建一次, 显隐靠 `display`)
5. **空文字防呆**: 文字为空时确认按钮 disabled

## 设计 (浮层 + autofocus)

| 选项 | 优势 | 劣势 |
|---|---|---|
| **模态浮层 + textarea** (选) | 焦点明确, ESC 关闭, 跟 macOS 习惯一致 | 阻挡背景 (设计: 80% 透明, 背景能看见) |
| 行内 inline 编辑 | 不阻挡 | 3D 空间里文字位置难选 |
| Command palette (cmd+k) | 高级用户友好 | 学习曲线, 不适合"主入口" |

选模态浮层理由: **首屏入口必须零学习成本**。

## 接口 (待实做)

```js
import { QuickAddPanel } from './interaction/quick-add-panel.js';

const panel = new QuickAddPanel({
  layers,                    // 从 LayerSystem.getLayers() 拿
  onAdd: async ({ text, layerId, tags }) => {
    // 委托给 action-router
    return actionRouter.dispatch({
      type: 'ADD_THOUGHT',
      payload: { text, layerId, tags, createdBy: 'user' }
    });
  },
});
panel.attach();              // 挂载到主视图右上
panel.toggle();              // 切显隐
panel.focusInput();          // 显示时自动 focus textarea

// 全局快捷键
window.addEventListener('keydown', (e) => {
  if (e.key === 'n' && !e.metaKey && !e.ctrlKey && !isInputFocused()) {
    e.preventDefault();
    panel.toggle();
  }
});
```

## 关键决策 (待沉淀到 @note)

1. **3 步流程**: 按钮 → textarea (autofocus) → layer select (默认 con-middle)
2. **快捷键 `N`**: 跟 macOS Notion / Obsidian 习惯一致, 不用 cmd+k
3. **浮层位置右上**: 跟 S2.11 DebugOverlay 不冲突 (debug 放左下, quick-add 放右上)
4. **回车提交, ESC 关闭**: 跟 macOS 模态一致
5. **错误反馈**: action-router 返错时浮层底部红字 (3 秒后淡出)

| 风险 | 等级 | 缓解 |
