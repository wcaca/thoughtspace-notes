# S2.17 Phase Color Lerp — 相变颜色 lerp (SEED 灰 → 真温度色)

> 状态: 计划 (2026-07-20)
> 配套: `src/v2/render/thought-mesh.js` (新增 _computePhaseColorMod + 改 _writeInstance) + `tests/v2/phase-color-lerp.test.js` (新)
> 集成: 跟 S2.14 scale + S2.15 alpha + S2.16 ease-out 串联, 4 维度同步
> 上游: S2.6 温度色 (temperatureToColor) + S2.14 phase-transition + S2.15 alpha + S2.16 easing

## WHY (为什么)

S2.14 + S2.15 + S2.16 解决了 scale/alpha/缓动, 但 **颜色还是直接用真温度色**:
- SEED 起步: `currentPhase=SEED, progress=0` → thought-mesh 直接渲染温度色 (蓝/红/中性)
- 用户视觉: "念头凭空出现, 颜色已经是真色, 缺少'从灰→彩'的渐变感"

7-19 backlog 写: "S2.17 颜色 lerp (温度色从灰→彩)"

**S2.17 = 视觉补完**:
- **SEED 起步**: color = GRAY (灰), progress → 1 过程中, color lerp 到真温度色
- **相变中** (CRYSTAL→MEMORY): 颜色已经是真色, 不需要再 lerp
- **已完成**: color = 真温度色 (无变化)
- **复用 S2.16 缓动**: 颜色 lerp 跟 scale/alpha 走同一个 easeOutCubic, 4 维度同步 (scale + alpha + 颜色 + 缓动)

## 现状 (问题点)

```js
// thought-mesh.js:_writeInstance (S2.6)
// 8. 颜色（温度色）
this._tempColor.copy(temperatureToColor(temperature));
// 直接写入, 没有 phase-aware 调整
```

`temperatureToColor(0.5)` 返回 TEMP_NEUTRAL (0xc9d6ea 浅蓝灰), 但 `temperatureToColor(0)` 是 TEMP_COLD (0x4a90e2 蓝), `temperatureToColor(1)` 是 TEMP_WARM (0xe94560 红).

如果用户加的念头 temperature=0.8 (热红), SEED 起步时已经显示成红, 跟"念头刚出现"的感觉不匹配.

## 修复方案

### 1. 新常量: PHASE_GRAY (固定灰色, lerp 起点)

```js
const PHASE_GRAY_COLOR = new THREE.Color(0x808080);  // 中性灰
```

### 2. _computePhaseColorMod helper (纯函数, 易测)

```js
/**
 * S2.17: 相变颜色 lerp (从灰 → 真温度色)
 *   - SEED 起步 (progress=0): color = GRAY (灰)
 *   - SEED 中 (0<progress<1): color lerp(GRAY, 真温度色, easedProg)
 *   - 其他相变中 (其他 phase, 0<progress<1): color = 真温度色 (不 lerp, 已经是真色)
 *   - 已完成 (progress=1): color = 真温度色
 * @private
 * @param {Thought} thought
 * @param {THREE.Color} trueColor - 真温度色 (由 temperatureToColor 算出)
 * @returns {THREE.Color} 新的 Color (caller 负责 setHex 或 copy)
 */
_computePhaseColorMod(thought, trueColor) {
  const currentPhase = thought._transient?.currentPhase ?? ThoughtPhase.SEED;
  const linearProg = thought._transient?.phaseTransitionProgress ?? 0;
  const phaseProg = this._applyPhaseEasing(linearProg);  // S2.16 缓动共享
  
  // SEED phase 才需要 lerp (其他 phase 颜色已经是真色)
  if (currentPhase === ThoughtPhase.SEED && linearProg < 1) {
    // gray → trueColor, lerp 系数 = phaseProg
    return trueColor.clone().lerp(PHASE_GRAY_COLOR, 1 - phaseProg);
    // 当 phaseProg=0: lerp(trueColor, gray, 1) = gray ✓
    // 当 phaseProg=1: lerp(trueColor, gray, 0) = trueColor ✓
  }
  return trueColor.clone();  // 非 SEED 或已完成, 直返真色
}
```

注: `Color.lerp(other, alpha)` = `this * (1 - alpha) + other * alpha`. 所以 `lerp(trueColor, gray, 1 - phaseProg)` 等价于 `trueColor * phaseProg + gray * (1 - phaseProg)`. ✓

### 3. _writeInstance 改用 _computePhaseColorMod

```js
// 替换原 _tempColor.copy(temperatureToColor(temperature))
const trueColor = temperatureToColor(temperature);
const finalColor = this._computePhaseColorMod(thought, trueColor);
this._tempColor.copy(finalColor);
```

## 测试计划 (10 测试)

`tests/v2/phase-color-lerp.test.js` (新, ~180 行):

- **4 个 _computePhaseColorMod 基础测试**:
  - SEED + progress=0 → 纯灰 (RGB ≈ 0.502, 0.502, 0.502)
  - SEED + progress=1 → 真温度色 (跟 trueColor 一样)
  - CRYSTAL + progress=0.5 → 真温度色 (非 SEED, 不 lerp)
  - MEMORY + progress=0.5 → 真温度色 (非 SEED, 不 lerp)
- **3 个 _computePhaseColorMod 缓动串联测试**:
  - SEED + progress=0.5 → gray*0.177 + trueColor*0.823 (eased)
  - SEED + progress=0.1 → gray*0.768 + trueColor*0.232 (eased 前段快)
  - SEED + progress=0.9 → gray*0.001 + trueColor*0.999 (eased 后段慢, 几乎全色)
- **1 个 跟 _computePhaseScaleMod 同步测试**:
  - SEED + progress=0.5 → color lerp 系数 = scale 缓动值 (eased 0.823)
- **2 个 _writeInstance 集成测试**:
  - temperature=0 (TEMP_COLD, 蓝) + SEED+progress=0 → 灰色 (非蓝色)
  - temperature=0 (TEMP_COLD, 蓝) + SEED+progress=1 → 真蓝 (0.29, 0.565, 0.886)
