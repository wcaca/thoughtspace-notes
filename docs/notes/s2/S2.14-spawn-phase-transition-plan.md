## WHY (为什么)

S2.13 QuickAddPanel 落地后, 用户可以 "点 + → 输文字 → 选层 → 确认" 加念头.
但 **新念头出现是瞬移** (spawnSampleThought 调 `t.tickPhaseTransition(CRYSTAL)` 直接跳 progress=1).

7-16 backlog 写: "S2.14 phase-transition 入场动画 (quick-add 后"念头从无到有"动画)"

**S2.14 = 修复 + 视觉**:
- **修复**: spawn 改用 `startPhaseTransition(CRYSTAL)` (从 SEED 起步, 0.8s 推进)
- **视觉**: thought-mesh 读 `currentPhase === SEED` → scale=0 (隐); progress→1 过程中 scale 0→1 (爆裂出现)
- **集成**: SEED 态的 thought-mesh 实例不参与 raycast (空 mesh 不可点)

## 现状 (问题点)

```js
// main.js:223
t.tickPhaseTransition(ThoughtPhase.CRYSTAL);  // 直接 progress=1, 无动画
```

虽然 thought-mesh._writeInstance 读 `phaseProgress` 算 `phaseScaleMod = 0.7 + 0.3*phaseProg`,
但因为 progress 永远是 1, **实际从未<1**, 所以 `phaseScaleMod` 永远是 1.0,
没有"收缩感"的视觉差异.

且 `currentPhase` 默认是 SEED (DEFAULT_THOUGHT_CONFIG.phase), 视觉上 SEED 跟 CRYSTAL 一样,
没区分.

## 修复方案

### 1. main.js: spawn 改用 startPhaseTransition

```js
function spawnSampleThought(opts = {}) {
  const t = new Thought({
    content: opts.content || '默认念头',
    layerId: opts.layerId || defaultLayer,
    position: opts.position || { vertical: 0.5, radial: 0.4, orbital: Math.random() * Math.PI * 2 },
    space,
    layerSystem,
  });
  // 播 SEED → CRYSTAL 入场动画 (800ms, 由 render-pipeline tickPhaseTransitions 推进)
  t.startPhaseTransition(ThoughtPhase.CRYSTAL);
  thoughtRefs.set(t.id, t);
  thoughtMesh.upsert(t, { viewVertical: 0.5 });
  return t;
}
```

### 2. thought-mesh: SEED 态视觉 (scale=0 隐, 进度 0→1 缩放弹出)

```js
// _writeInstance 内, 替换 phaseScaleMod 逻辑:
const currentPhase = thought._transient?.currentPhase ?? ThoughtPhase.SEED;
const phaseProg = thought._transient?.phaseTransitionProgress ?? 0;
let phaseScaleMod;
if (currentPhase === ThoughtPhase.SEED) {
  // SEED 起步: scale=0, 进度 → 1 时 scale → 1.0 (爆裂出现)
  phaseScaleMod = phaseProg;
} else if (phaseProg < 1) {
  // 其他相变中: 0.7 → 1.0 收缩感
  phaseScaleMod = 0.7 + 0.3 * phaseProg;
} else {
  phaseScaleMod = 1.0;
}
const finalScale = displayScale * phaseScaleMod;
```

### 3. 透明度 (alpha) 渐入 (可选)

`aPhaseProgress` 已经作为 instanced attribute 上传, 但 shader 没用它做 alpha.
本期 S2.14 先不动 shader (避免复杂), 视觉走 scale 弹出 + 颜色 (温度色本来就有).

如果验收发现"scale=0 → 1"太单调, 再补 alpha shader 渐入.

## 接口 (待实做 + 验证)

```js
// 1. spawn 真实播动画
const t = spawnSampleThought({ content: 'test' });
assert(t._transient.targetPhase === ThoughtPhase.CRYSTAL);
assert(t._transient.phaseTransitionProgress === 0);  // 起步 0, 不是 1

// 2. tick 推进
thoughtMesh.tickPhaseTransitions(0.4);  // 推 400ms
assert(t._transient.phaseTransitionProgress === 0.5);  // 0.4/0.8
thoughtMesh.tickPhaseTransitions(0.4);
assert(t._transient.phaseTransitionProgress === 1.0);
assert(t._transient.currentPhase === ThoughtPhase.CRYSTAL);

// 3. thought-mesh 视觉: SEED 时 scale=0
const { scale } = thoughtMesh._getRenderedScale(t, { viewVertical: 0.5 });
assert(scale === 0);  // SEED 起步
thoughtMesh.tickPhaseTransitions(0.5);
const { scale: scaleMid } = thoughtMesh._getRenderedScale(t, { viewVertical: 0.5 });
assert(scaleMid > 0 && scaleMid < 1);  // 推进中
```
